## 핵심 개념 상세 정리

### 6-1. 금융 데이터 스키마 설계 — @dataclass

Python 3.7+에서 도입. `@dataclass` 데코레이터를 붙이면 `__init__`, `__repr__`, `__eq__` 를 자동 생성.
`__post_init__()` 을 정의하면 인스턴스 생성 직후 추가 로직(유효성 검증 등)을 실행할 수 있음.

#### @dataclass 기본 구조

```python
from dataclasses import dataclass, field, asdict
import json

@dataclass
class ETFSchema:
    ticker: str               # 종목 코드 (예: "069500")
    name: str                 # ETF 이름
    category: str             # 카테고리 (국내주식·해외주식·채권 등)
    expense_ratio: float      # 총보수율 (%)
    dividend_yield: float     # 배당수익률 (%)
    keywords: list = field(default_factory=list)  # 특징 키워드

    def __post_init__(self):
        # expense_ratio 범위 검증
        if not (0 <= self.expense_ratio <= 5):
            raise ValueError(f"expense_ratio 범위 오류: {self.expense_ratio}")
```

#### JSON 직렬화 / 역직렬화

```python
etf = ETFSchema(ticker="069500", name="KODEX 200", category="국내주식",
                expense_ratio=0.15, dividend_yield=1.2, keywords=["코스피","지수추종"])

# 직렬화: dataclass → dict → JSON 문자열
data = asdict(etf)                          # {'ticker': '069500', ...}
json_str = json.dumps(data, ensure_ascii=False)

# 역직렬화: JSON 문자열 → dict → dataclass 인스턴스
loaded = ETFSchema(**json.loads(json_str))
```

#### ETFSchema vs ETFReturns 비교

| 클래스 | 검증 대상 | 범위 | 목적 |
|---|---|---|---|
| `ETFSchema` | `expense_ratio` | 0 ~ 5 % | ETF 메타정보 구조화 |
| `ETFReturns` | `daily_return` | -100 ~ 500 % | 수익률 데이터 유효성 확인 |

### 6-3. 결측치 처리 — ffill · interpolate · rolling

금융 데이터는 주말·공휴일·거래 정지로 결측값이 발생. 처리 방식에 따라 분석 결과가 달라짐.

#### 세 방식 비교

| 방법 | 원리 | 적합 상황 | 주의 |
|---|---|---|---|
| `ffill()` | 직전 유효값을 그대로 복사 | 짧은 결측, 단계형 가격 | 장기 결측 시 왜곡 |
| `interpolate('linear')` | 앞뒤 값을 선형 보간 | 완만한 추세가 예상될 때 | 급등락 구간엔 부적합 |
| `rolling(N, min_periods=1).mean()` | 주변 N개 평균 | 장기 결측, 노이즈 완화 | N 크기에 따라 평활화 정도 다름 |

```python
df_ffill = df['Close'].fillna(method='ffill')
df_interp = df['Close'].interpolate(method='linear')
df_rolling = df['Close'].fillna(df['Close'].rolling(5, min_periods=1).mean())
```

### 6-4. 이상치 탐지 — IQR · Z-score

비정상적인 가격 움직임(오류 데이터, 극단값)을 탐지해 분석 왜곡 방지.

#### IQR (사분위 범위)

```python
Q1 = df['Close'].quantile(0.25)
Q3 = df['Close'].quantile(0.75)
IQR = Q3 - Q1

lower = Q1 - 1.5 * IQR   # 하한선
upper = Q3 + 1.5 * IQR   # 상한선

iqr_mask = (df['Close'] >= lower) & (df['Close'] <= upper)  # 정상 범위 마스크
```

#### Z-score

```python
mean = df['Close'].mean()
std  = df['Close'].std()

z_scores = (df['Close'] - mean) / std
z_mask = z_scores.abs() <= 3  # ±3σ 이내를 정상으로 간주
```

#### 두 방식 결합

| 연산자 | 의미 | 효과 |
|---|---|---|
| `iqr_mask & z_mask` | 둘 다 정상인 것만 유지 | 보수적 — 이상치 제거 범위 좁음 |
| `iqr_mask \| z_mask` | 하나라도 정상이면 유지 | 공격적 — 더 많이 제거 |

> 실무: 금융 데이터는 `&` 결합(보수적)이 기본. 오류 제거가 목적이면 `|` 사용 검토.

### 6-5. 특성 추출 & 전처리 파이프라인

수집한 종가 데이터에서 분석에 필요한 파생 특성을 추출하고, 전처리 단계를 함수로 통합.

#### 주요 파생 특성

```python
df['return']     = df['Close'].pct_change()                        # 일별 수익률 (%)
df['volatility'] = df['return'].rolling(5).std()                   # 5일 변동성
df['vol_20']     = df['return'].rolling(20).std()                  # 20일 변동성
df['log_return'] = np.log(df['Close'] / df['Close'].shift(1))      # 로그 수익률
```

#### 전처리 파이프라인 함수

```python
def preprocess_pipeline(df: pd.DataFrame) -> pd.DataFrame:
    result = df.copy()
    # 1단계: 결측치 보간 (선형)
    result['Close'] = result['Close'].interpolate(method='linear')
    # 2단계: IQR 이상치 제거
    Q1, Q3 = result['Close'].quantile([0.25, 0.75])
    IQR = Q3 - Q1
    mask = (result['Close'] >= Q1 - 1.5*IQR) & (result['Close'] <= Q3 + 1.5*IQR)
    result = result[mask]
    # 3단계: 특성 추가
    result['return']     = result['Close'].pct_change()
    result['volatility'] = result['return'].rolling(5).std()
    result['log_return'] = np.log(result['Close'] / result['Close'].shift(1))
    return result.dropna()
```

### 6-6. 시계열 분석 — SMA · EMA · Trend Decomposition

주가 시계열에서 추세를 파악하고 계절성·잔차를 분리하는 기법.

#### SMA vs EMA 비교

| 방식 | 계산 | 가중치 | 민감도 | 적합 상황 |
|---|---|---|---|---|
| SMA (단순이동평균) | `rolling(N).mean()` | 모든 기간 동일 | 낮음 (지연 큼) | 장기 추세 확인 |
| EMA (지수이동평균) | `ewm(span=N).mean()` | 최근일수록 높음 | 높음 (지연 작음) | 단기 추세 전환 포착 |

```python
df['SMA_20'] = df['Close'].rolling(window=20).mean()   # 20일 단순이동평균
df['EMA_20'] = df['Close'].ewm(span=20, adjust=False).mean()  # 20일 지수이동평균
```

#### Trend Decomposition — seasonal_decompose

```python
from statsmodels.tsa.seasonal import seasonal_decompose

result = seasonal_decompose(df['Close'].dropna(),
                             model='additive',   # 'additive' 또는 'multiplicative'
                             period=20)          # 주기 (거래일 기준 약 1개월)

# result.trend    — 장기 추세 성분
# result.seasonal — 주기 성분
# result.resid    — 잔차 (= 실제값 - 추세 - 계절성)
result.plot()
```

> `additive`: 실제값 = 추세 + 계절성 + 잔차 (가격 변동 폭이 일정할 때)
> `multiplicative`: 실제값 = 추세 × 계절성 × 잔차 (변동 폭이 추세에 비례할 때)

### 6-7. 멀티자산 비교 & 성과 지표 (Sharpe · MDD)

여러 ETF를 동시에 수집·비교하고, 투자 관점에서 의미 있는 성과 지표를 계산.

#### 멀티자산 수집 & 누적수익률

```python
tickers = ["069500", "114800", "139660", "308620"]
prices = {}
for t in tickers:
    prices[t] = fdr.DataReader(t, "2023-01-01")['Close']

price_df  = pd.DataFrame(prices)           # 종가 합산 DataFrame
returns   = price_df.pct_change().dropna() # 일별 수익률
cum_ret   = (1 + returns).cumprod()        # 누적수익률 (기준: 1.0)
```

#### 성과 지표 계산

| 지표 | 수식 | 의미 |
|---|---|---|
| 연환산 수익률 | `mean_daily × 252` | 하루 평균 수익률 × 연간 거래일 |
| 연환산 변동성 | `std_daily × √252` | 하루 변동성을 연간 단위로 환산 |
| Sharpe Ratio | `ann_return / ann_vol` | 위험 1단위당 초과 수익 (높을수록 좋음) |
| MDD | `min((peak - current) / peak)` | 최대 낙폭 — 고점 대비 최대 하락률 |

```python
def compute_performance(returns: pd.Series) -> dict:
    ann_ret = returns.mean() * 252
    ann_vol = returns.std() * (252 ** 0.5)
    sharpe  = ann_ret / ann_vol if ann_vol > 0 else 0

    cum = (1 + returns).cumprod()
    peak = cum.cummax()
    mdd  = ((cum - peak) / peak).min()   # 음수값, 절댓값이 클수록 손실 큼

    return {"ann_return": ann_ret, "ann_volatility": ann_vol,
            "sharpe": sharpe, "mdd": mdd}
```

#### 상관관계 분석

```python
corr_matrix = returns.corr()  # Pearson 상관계수 행렬 (−1 ~ 1)
# 1에 가까울수록 같은 방향, −1에 가까울수록 반대 방향, 0은 무관계
```

### 6-8. LLM 기반 분석 리포트 & DataFrame 병합

#### 계산형 리포트 vs LLM-only 리포트

| 방식 | 함수 | 특징 | 한계 |
|---|---|---|---|
| 계산형 + LLM | `generate_analysis_report()` | 수치(SMA·변동성·Z-score·왜도·첨도)를 계산한 뒤 LLM이 해석 | 코드 복잡도 높음 |
| LLM-only | `generate_analysis_report_llm_only()` | 가격 리스트만 전달 → LLM이 직접 분석 | 수치 정밀도 낮음, 할루시네이션 위험 |

```python
def generate_analysis_report(ticker: str, prices: list, llm) -> str:
    sma_5  = pd.Series(prices).rolling(5).mean().iloc[-1]
    vol_5  = pd.Series(prices).pct_change().rolling(5).std().iloc[-1]
    z_last = (prices[-1] - np.mean(prices)) / np.std(prices)
    skew   = pd.Series(prices).skew()
    kurt   = pd.Series(prices).kurt()

    prompt = f"""종목: {ticker}
현재가: {prices[-1]:.0f}  SMA5: {sma_5:.0f}  변동성(5일): {vol_5:.4f}
Z-score: {z_last:.2f}  왜도: {skew:.2f}  첨도: {kurt:.2f}
위 지표를 바탕으로 투자 관점 분석을 작성하시오."""
    return llm.invoke(prompt).content
```

### 6-11. BM25 (Kiwi 형태소 분석) & 메타데이터 필터 검색

#### Kiwi 한국어 형태소 분석

```python
from kiwipiepy import Kiwi
from rank_bm25 import BM25Okapi

kiwi = Kiwi()

def tokenize_ko(text: str) -> list[str]:
    # 형태소 단위로 분리 — 고유명사(NNP)·일반명사(NNG) 중심 추출
    tokens = kiwi.tokenize(text)
    return [t.form for t in tokens if t.tag in ('NNP', 'NNG', 'SL', 'SN')]

# 코퍼스 토큰화
tokenized_corpus = [tokenize_ko(doc.page_content) for doc in documents]
bm25 = BM25Okapi(tokenized_corpus)
```

#### BM25 파라미터

| 파라미터 | 기본값 | 역할 |
|---|---|---|
| `k1` | 1.5 | 단어 빈도(TF) 포화 계수. 높을수록 빈도 영향 증가 |
| `b` | 0.75 | 문서 길이 정규화 계수. 1에 가까울수록 길이 보정 강함 |

```python
def bm25_search(query: str, bm25, documents: list, top_k: int = 5) -> list:
    query_tokens = tokenize_ko(query)
    scores = bm25.get_scores(query_tokens)               # 문서별 BM25 점수
    top_idx = sorted(range(len(scores)), key=lambda i: scores[i], reverse=True)[:top_k]
    return [(documents[i], scores[i]) for i in top_idx]
```

#### 메타데이터 필터 검색

```python
def filtered_search(vectorstore, query: str,
                    filters: dict, k: int = 5, fetch_k: int = 30):
    # fetch_k개를 먼저 벡터 검색한 뒤 조건 필터링 → 최종 k개 반환
    candidates = vectorstore.similarity_search_with_score(query, k=fetch_k)
    result = []
    for doc, score in candidates:
        meta = doc.metadata
        match = True
        for field, cond in filters.items():
            val = meta.get(field)
            if 'less_than'    in cond and not (val < cond['less_than']):    match = False
            if 'greater_than' in cond and not (val > cond['greater_than']): match = False
            if 'eq'           in cond and val != cond['eq']:                match = False
        if match:
            result.append((doc, score))
    return result[:k]
```

### 6-12. 스마트 필터 & 하이브리드 검색

#### 스마트 필터 — 자연어 → JSON 조건 추출

```python
def extract_filters(query: str, llm) -> dict:
    prompt = f"""사용자 쿼리에서 ETF 검색 조건을 JSON으로 추출하시오.
가능한 필드: expense_ratio(총보수율), category(카테고리), risk_level(위험등급)
조건 형식: {{"expense_ratio": {{"less_than": 0.5}}, "category": {{"eq": "국내주식"}}}}

쿼리: "{query}"
JSON만 출력:"""
    raw = llm.invoke(prompt).content.strip()
    return json.loads(raw.strip('```json').strip('```'))

def smart_document_search(query: str, vectorstore, llm, k: int = 5):
    filters = extract_filters(query, llm)          # 자연어 → 필터 조건 추출
    return filtered_search(vectorstore, query, filters, k=k)
```

#### 하이브리드 검색 — Vector + BM25 가중 결합

```python
def hybrid_search(query, vectorstore, bm25, documents,
                  alpha: float = 0.5, k: int = 5) -> list:
    # 벡터 검색
    vec_results = vectorstore.similarity_search_with_score(query, k=k*3)
    # BM25 검색
    bm25_results = bm25_search(query, bm25, documents, top_k=k*3)

    # min-max 정규화 후 가중 합산
    def normalize(scores):
        mn, mx = min(scores), max(scores)
        return [(s - mn) / (mx - mn + 1e-9) for s in scores]

    vec_scores  = normalize([s for _, s in vec_results])
    bm25_scores = normalize([s for _, s in bm25_results])

    combined = {}
    for (doc, _), vs in zip(vec_results, vec_scores):
        combined[doc.metadata['ticker']] = {'doc': doc, 'score': alpha * vs}
    for (doc, _), bs in zip(bm25_results, bm25_scores):
        t = doc.metadata['ticker']
        if t in combined:
            combined[t]['score'] += (1 - alpha) * bs
        else:
            combined[t] = {'doc': doc, 'score': (1 - alpha) * bs}

    ranked = sorted(combined.values(), key=lambda x: x['score'], reverse=True)
    return [(r['doc'], r['score']) for r in ranked[:k]]
```

> `alpha = 1.0` → 순수 벡터 검색, `alpha = 0.0` → 순수 BM25
> 최적 alpha는 Hit Rate / MRR로 정량 평가해 결정

### 6-13. 검색 성능 평가 — Hit Rate@K · MRR

#### 평가 쿼리셋 구성 — Synthetic Query

```python
def generate_synthetic_queries(etf_knowledge_base: str, llm, n: int = 5) -> list[str]:
    prompt = f"""아래 ETF 정보를 바탕으로 투자자가 물어볼 법한 쿼리 {n}개를 생성하시오.

{etf_knowledge_base}

```

#### Hit Rate@K

K개 검색 결과 안에 정답 문서가 **하나라도** 포함되면 1, 없으면 0. 가장 단순한 이진 메트릭.

```python
def hit_rate_at_k(retrieved: list[str], relevant: set[str], k: int) -> int:
    return int(bool(set(retrieved[:k]) & relevant))

# 여러 쿼리에 대한 평균
avg_hit_rate = sum(hit_rate_at_k(r, rel, k=5)
                   for r, rel in query_results) / len(query_results)
```

#### MRR (Mean Reciprocal Rank)

정답이 **몇 번째 위치**에 나왔는지까지 반영. 순위가 높을수록 점수 높음.

```python
def mrr_at_k(retrieved: list[str], relevant: set[str], k: int) -> float:
    for rank, doc_id in enumerate(retrieved[:k], start=1):
        if doc_id in relevant:
            return 1.0 / rank   # 정답 첫 등장 위치의 역수
    return 0.0

avg_mrr = sum(mrr_at_k(r, rel, k=5) for r, rel in query_results) / len(query_results)
```

| 메트릭 | 정답 위치 고려 | 정답 수 고려 | 주요 용도 |
|---|---|---|---|
| Hit Rate@K | ✗ (있으면 1) | ✗ | 기본 커버리지 확인 |
| MRR@K | ✓ (순위 반영) | ✗ (첫 정답만) | 추천 시스템, 검색 엔진 |

### 6-14. Reranking (MMR · LLM Listwise) & 콘텐츠 기반 필터링 (CBF)

#### MMR (Maximum Marginal Relevance)

관련성과 다양성을 동시에 최적화. 이미 선택된 문서와 너무 유사한 문서를 패널티.

```
MMR 점수 = λ × sim(query, d) − (1−λ) × max sim(d, selected_docs)
```

```python
# LangChain 내장
results = vectorstore.max_marginal_relevance_search(
    query, k=5, fetch_k=20, lambda_mult=0.5
)
# lambda_mult=0.7~0.9: 관련성 우선, lambda_mult=0.3~0.5: 다양성 우선
```

#### LLM Listwise Reranking

```python
def llm_listwise_rerank(query: str, candidates: list, llm, top_k: int = 5) -> list:
    doc_list = "\n".join(
        f"[{i+1}] {doc.metadata['ticker']}: {doc.page_content[:100]}"
        for i, (doc, _) in enumerate(candidates)
    )
    prompt = f"""쿼리: "{query}"
다음 ETF 목록을 쿼리와의 관련성 순으로 재정렬하여 번호만 JSON 배열로 반환:
{doc_list}
예: [3,1,5,2,4]"""
    order = json.loads(llm.invoke(prompt).content.strip())
    return [candidates[i-1] for i in order[:top_k]]
```

#### 콘텐츠 기반 필터링 (CBF)

협업 필터링(Collaborative Filtering)과 달리 **아이템 속성 자체**를 기반으로 유사도 계산 → 사용자 행동 데이터 없이도 추천 가능(cold start 강점).

```python
from sklearn.metrics.pairwise import cosine_similarity
import numpy as np

def etf_to_vector(etf_meta: dict) -> np.ndarray:
    risk_map = {"low": 0, "medium-low": 1, "medium": 2, "high": 3}
    cat_map   = {"국내주식": 0, "해외주식": 1, "채권": 2, "혼합": 3, "기타": 4}
    return np.array([
        etf_meta['expense_ratio'],
        etf_meta['dividend_yield'],
        risk_map.get(etf_meta['risk_level'], 2),
        cat_map.get(etf_meta['category'], 4),
    ])

def build_item_similarity(etfs: list[dict]) -> np.ndarray:
    vectors = np.array([etf_to_vector(e) for e in etfs])
    return cosine_similarity(vectors)  # N×N 유사도 행렬 (0~1)

def cbf_similar_items(ticker: str, etfs: list, sim_matrix: np.ndarray,
                      k: int = 5) -> list:
    idx = next(i for i, e in enumerate(etfs) if e['ticker'] == ticker)
    scores = sim_matrix[idx]
    # 자기 자신 제외 후 상위 k개 반환
    ranked = sorted(range(len(scores)), key=lambda i: scores[i], reverse=True)
    return [etfs[i] for i in ranked if i != idx][:k]

def cbf_diverse(ticker: str, etfs: list, sim_matrix: np.ndarray,
                k: int = 5) -> list:
    # 카테고리 다양성 보장: 같은 카테고리에서 최대 1개만 포함
    candidates = cbf_similar_items(ticker, etfs, sim_matrix, k=k*3)
    seen_cats, result = set(), []
    for etf in candidates:
        if etf['category'] not in seen_cats:
            result.append(etf)
            seen_cats.add(etf['category'])
        if len(result) >= k:
            break
    return result
```

## 고려사항 & 질문 모음

1. **`@dataclass` vs `dict`**: dict는 키 오타 검증이 없고 `__post_init__` 같은 훅이 없음. dataclass는 타입 힌트·IDE 자동완성·유효성 검증·`asdict()` 직렬화를 한꺼번에 제공 → 협업·유지보수 코드에서 유리.

2. **`ffill` vs `interpolate` 선택 기준**: `ffill`은 마지막 유효값을 계단형으로 복사해 "실제 거래가 없었다"는 사실을 그대로 반영. `interpolate`는 누락 기간 동안 부드럽게 변화했다고 가정. 장기 거래 중지(상장 폐지 직전 등)엔 ffill, 단기 데이터 오류엔 interpolate가 적합.

3. **IQR과 Z-score를 `&`로 결합하는 이유**: `|`(합집합)로 결합하면 한 쪽 기준만 통과해도 이상치로 분류해 실제 가격 급등락도 제거될 수 있음. 금융 데이터에서 단기 이벤트(배당락일, 서킷브레이커)는 정상 범위 이탈일 수 있으므로 둘 다 이상치로 판단할 때만 제거하는 `&` 방식이 안전.

4. **SMA와 EMA의 지연(lag) 차이**: SMA는 N일 전~현재에 동일 가중치를 줘 반응이 N/2일 지연됨. EMA는 최근 데이터에 더 높은 가중치를 부여해 지연이 적고 추세 전환을 빠르게 포착. 단, EMA는 급등락에 과민 반응할 수 있음.

5. **계산형 LLM 리포트 vs LLM-only 리포트**: 계산형은 정확한 수치(Z-score 2.3, 변동성 0.018 등)를 Python으로 계산한 뒤 LLM에 전달 → 해석 정밀도 높음. LLM-only는 가격 목록만 전달해 LLM이 직접 패턴을 추론하므로 수치 계산 오류 발생 가능. 프로덕션에서는 계산형을 기본으로 사용.

6. **FAISS와 BM25의 역할 분담**: FAISS(벡터 검색)는 "의미적으로 비슷한 것" 을 찾는 데 강함(예: "안정적 수익" → 변동성 낮은 채권 ETF). BM25는 고유명사·코드 등 **정확한 키워드 매칭**에 강함(예: "KODEX 200" → 해당 ETF 정확히 반환). 두 방식을 alpha로 가중 결합하면 양쪽 강점을 모두 활용.

7. **Hit Rate@K vs MRR@K 선택**: Hit Rate는 "결과 K개 안에 관련 항목이 있는가"만 확인해 추천 순서를 무시. MRR은 관련 항목의 **등장 순위**를 반영해 1위 추천 정확도를 더 잘 측정. 사용자가 상위 1~2개 결과만 클릭하는 서비스라면 MRR이 더 적합한 지표.

8. **CBF vs Collaborative Filtering의 cold start 문제**: Collaborative Filtering은 사용자 행동 로그(구매·클릭)가 충분해야 정확도가 높음 → 신규 아이템(상장한 지 얼마 안 된 ETF)은 데이터 부족으로 추천 불가(cold start). CBF는 아이템 속성(카테고리·보수율·위험등급)만으로 유사도를 계산하므로 신규 ETF도 즉시 추천 가능. 단, 사용자 취향의 다양성을 발굴하는 데는 Collaborative가 유리.
