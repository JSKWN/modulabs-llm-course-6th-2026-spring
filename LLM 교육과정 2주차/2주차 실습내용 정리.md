## 핵심 개념 상세 정리

### 4-1. LLM API 기초 (ChatOpenAI & LCEL)

#### ChatOpenAI 기본 사용

```python
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

llm = ChatOpenAI(model="gpt-4o-mini")

prompt = ChatPromptTemplate.from_messages([
    ("system", "당신은 {role} 전문가입니다."),
    ("user", "{question}")
])

parser = StrOutputParser()
chain = prompt | llm | parser

result = chain.invoke({"role": "청약", "question": "청약 자격은?"})
```

#### LCEL (LangChain Expression Language) 핵심 개념

- **파이프라인 구성**: `A | B | C` — 앞 컴포넌트의 출력이 다음 입력으로 전달
- **`invoke`**: 단일 동기 호출
- **`stream`**: 청크 단위 스트리밍 (토큰 하나씩 출력)
- **`RunnablePassthrough`**: 입력을 그대로 다음 단계에 전달 (question 전달 시 사용)
- **`RunnableParallel`**: 여러 체인을 병렬 실행 후 dict로 반환

```python
# stream 예시
for chunk in chain.stream({"context": ctx, "question": q}):
    print(chunk, end="", flush=True)
```

#### 메시지 타입

| 타입 | 클래스 | 역할 |
|------|--------|------|
| 시스템 | `SystemMessage` | AI의 역할/페르소나 정의 |
| 사용자 | `HumanMessage` | 유저의 질문 |
| AI | `AIMessage` | LLM의 응답 |

### 4-2. 임베딩 (Embeddings & 유사도)

#### OpenAIEmbeddings 기본

```python
from langchain_openai import OpenAIEmbeddings

embeddings = OpenAIEmbeddings(model="text-embedding-3-small")  # 1536차원

# 단일 쿼리 임베딩 (1D 리스트 반환)
query_vec = embeddings.embed_query("청약 자격 조건은?")
print(len(query_vec))  # 1536

# 복수 문서 임베딩 (2D 리스트 반환)
doc_vecs = embeddings.embed_documents(["문서1", "문서2", "문서3"])
print(len(doc_vecs), len(doc_vecs[0]))  # 3, 1536
```

#### embedQuery vs embedDocuments

| 메서드 | 입력 | 출력 | 용도 |
|--------|------|------|------|
| `embed_query(text)` | `str` | `list[float]` (1D) | 검색 쿼리 |
| `embed_documents(texts)` | `list[str]` | `list[list[float]]` (2D) | 문서 벡터화 |

> ⚠️ **주의**: `cosine_similarity`는 2D 입력을 요구하므로, query를 넘길 때는 `[query_vec]`로 감싸거나 reshape 필요

```python
from sklearn.metrics.pairwise import cosine_similarity

# cosine_similarity 2D 입력 필요
scores = cosine_similarity([query_vec], doc_vecs)  # shape: (1, N)
scores_1d = scores[0]  # shape: (N,) — 1D로 변환하여 사용
```

#### CacheBackedEmbeddings (임베딩 캐싱)

```python
from langchain_classic.embeddings import CacheBackedEmbeddings
from langchain_core.stores import InMemoryByteStore

store = InMemoryByteStore()
cached_embeddings = CacheBackedEmbeddings.from_bytes_store(
    embedding_model,
    store,
    namespace="embedding-cache"
)

# 동일 텍스트 재호출 시 캐시 사용 → 속도 향상, API 비용 절감
vecs1 = cached_embeddings.embed_documents(texts)  # API 호출
vecs2 = cached_embeddings.embed_documents(texts)  # 캐시에서 반환
```

> 💡 **ByteStore 종류**: `InMemoryByteStore` (메모리), `LocalFileStore` (파일), `RedisStore` (Redis)

### 4-3. RAG 개념과 DocumentStore

#### RAG vs Fine-tuning

| 항목 | RAG | Fine-tuning |
|------|-----|-------------|
| 학습 여부 | 없음 (검색 기반) | 모델 재학습 필요 |
| 최신 정보 반영 | 쉬움 (문서 추가) | 어려움 (재학습) |
| 비용 | 낮음 | 높음 |
| 적합한 경우 | 문서 기반 QA | 특정 스타일/도메인 적응 |

#### DocumentStore 직접 구현 (핵심 클래스)

```python
class DocumentStore:
    def __init__(self):
        self.documents = []
        self.next_id = 1

    def add(self, content, source="unknown"):
        self.documents.append({
            "id": self.next_id,
            "content": content,
            "source": source
        })
        self.next_id += 1

    def count(self):
        return len(self.documents)

    def search(self, keyword):
        return [doc for doc in self.documents if keyword in doc["content"]]

    def retrieve(self, query, top_k=3):
        """코사인 유사도 기반 벡터 검색"""
        if not self.documents:
            return []
        query_emb = embeddings.embed_query(query)
        contents = [doc["content"] for doc in self.documents]
        doc_embs = embeddings.embed_documents(contents)
        scores = cosine_similarity([query_emb], doc_embs)[0]
        sorted_docs = sorted(zip(self.documents, scores),
                             key=lambda x: x[1], reverse=True)
        return [(doc, float(score)) for doc, score in sorted_docs[:top_k]]

    def generate(self, query):
        """RAG: 검색 → 컨텍스트 구성 → LLM 답변 생성"""
        retrieved = self.retrieve(query, top_k=3)
        if not retrieved:
            return "관련 문서가 없습니다."
        contexts = "
".join([f"- {content}" for content, score in retrieved])
        messages = [
            SystemMessage(content="당신은 문서 기반 질문 답변 전문가입니다."),
            HumanMessage(content=f"{contexts}\n\n질문: {query}")
        ]
        response = llm.invoke(messages)
        return response.content
```

#### 핵심 Python 패턴 이해

**`sorted(zip(docs, scores), key=lambda x: x[1], reverse=True)`**

```python
docs = ["A", "B"]
scores = [0.1, 0.9]

# zip → [(A, 0.1), (B, 0.9)]
# sorted by x[1] (score) 내림차순 → [(B, 0.9), (A, 0.1)]
zipped = zip(docs, scores)
sorted_res = sorted(zipped, key=lambda x: x[1], reverse=True)
# → [("B", 0.9), ("A", 0.1)]

final = [(d, float(s)) for d, s in sorted_res]
```

#### TXT 파일 로딩

```python
from pathlib import Path

def load_text_files(directory: Path):
    documents = []
    for fp in sorted(directory.glob("*.txt")):  # *.txt 패턴 매칭
        text = fp.read_text(encoding="utf-8")
        doc = Document(
            page_content=text,
            metadata={
                "source": fp.name,
                "char_count": len(text)
            }
        )
        documents.append(doc)
    return documents
```

#### CSV 파일 로딩

```python
import csv

def load_csv_file(filepath):
    documents = []
    with open(filepath, "r", encoding="utf-8") as f:
        reader = csv.DictReader(f)
        for i, row in enumerate(reader):
            content = " | ".join(f"{k}:{v}" for k, v in row.items())
            doc = Document(
                page_content=content,
                metadata={"source": filepath.name, "row": i}
            )
            documents.append(doc)
    return documents
```

#### JSON 파일 로딩

```python
import json

def load_json_as_document(filepath):
    path = Path(filepath)
    with open(path, "r", encoding="utf-8") as f:
        data = json.load(f)
    content = json.dumps(data, ensure_ascii=False, indent=2)
    keys = list(data.keys())
    return Document(
        page_content=content,
        metadata={"source": path.name, "keys": keys}
    )
```

> 💡 **json.dumps vs json.dump**
> - `json.dumps(obj)` → Python 객체를 **문자열**로 변환 (s = string)
> - `json.dump(obj, f)` → Python 객체를 **파일**에 직접 저장

### 4-5. FAISS 벡터 DB

#### FAISS란?

- Facebook AI Research에서 개발한 고속 벡터 유사도 검색 라이브러리
- 인메모리 방식으로 빠른 검색, `save_local`/`load_local`로 영속성 지원
- RAG 파이프라인에서 문서 검색 엔진 역할

#### 벡터 DB 비교

| DB | 설치 | 특징 | 적합한 경우 |
|----|------|------|------------|
| **FAISS** | `pip install faiss-cpu` | 인메모리, 초고속 | 프로토타입, 소규모 |
| **Chroma** | `pip install chromadb` | 로컬 파일 저장 | 개발 환경 |
| **Qdrant** | Docker / API | HNSW 인덱스 | 중규모 프로덕션 |
| **Pinecone** | API | 완전 관리형 클라우드 | 대규모 프로덕션 |

#### FAISS 핵심 API

```python
from langchain_community.vectorstores import FAISS
from langchain_openai import OpenAIEmbeddings

embeddings = OpenAIEmbeddings(model="text-embedding-3-small")

# 1. 생성 — Document 리스트로부터 FAISS 인덱스 구축
vectorstore = FAISS.from_documents(docs, embeddings)

# 2-1. 유사도 검색 (상위 k개 Document 반환)
results = vectorstore.similarity_search("청약 자격은?", k=3)
for doc in results:
    print(doc.page_content)

# 2-2. 점수 포함 검색 (L2 거리 기반 — 낮을수록 유사)
results_with_score = vectorstore.similarity_search_with_score("청약 자격은?", k=3)
for doc, score in results_with_score:
    print(f"score={score:.4f}", doc.page_content[:40])

# 2-3. 벡터로 직접 검색 (API 호출 절약)
qvec = embeddings.embed_query("청약 자격은?")
results = vectorstore.similarity_search_by_vector(qvec, k=3)

# 2-4. MMR (다양성 고려 검색)
results = vectorstore.max_marginal_relevance_search(
    "청약 자격은?",
    k=3, fetch_k=10, lambda_mult=0.5
)

# 3. 저장 및 로드
vectorstore.save_local("faiss_index")
loaded = FAISS.load_local(
    "faiss_index",
    embeddings,
    allow_dangerous_deserialization=True  # .pkl 역직렬화 허용
)

# 4. 문서 추가
new_docs = [Document(page_content="새 FAQ 내용", metadata={"category": "청약"})]
vectorstore.add_documents(new_docs)
print(vectorstore.index.ntotal)  # 총 벡터 수 확인
```

### 4-6. Retriever & LCEL 심화

#### asRetriever로 Retriever 생성

```python
# similarity — 상위 k개 반환
retriever = vectorstore.as_retriever(
    search_type="similarity",
    search_kwargs={"k": 3}
)

# mmr — 다양성 고려 (중복 문서 방지)
retriever = vectorstore.as_retriever(
    search_type="mmr",
    search_kwargs={"k": 3, "fetch_k": 10, "lambda_mult": 0.5}
)

# similarity_score_threshold — 임계값 이상만 반환
retriever = vectorstore.as_retriever(
    search_type="similarity_score_threshold",
    search_kwargs={"score_threshold": 0.7, "k": 5}
)
```

> ⚠️ **주의**: `similarity_score_threshold` 사용 시 FAISS의 L2 거리가 0~1 범위를 벗어날 수 있어 경고 발생 가능. Cosine 기반 DB(Chroma 등)에서 더 안정적.

#### LCEL RAG 체인 완성 패턴

```python
from langchain_core.runnables import RunnablePassthrough, RunnableParallel
from langchain_core.output_parsers import StrOutputParser

# Document 리스트 → 텍스트 컨텍스트 문자열로 변환
def format_docs(docs):
    return "
".join(doc.page_content for doc in docs)

def extract_sources(docs):
    return [{"id": doc.id, "category": doc.metadata.get("category")} for doc in docs]

prompt = ChatPromptTemplate.from_messages([
    ("system", "다음 FAQ 문서를 참고하여 답변하세요:
{context}"),
    ("user", "{query}")
])

llm = ChatOpenAI(model="gpt-4o-mini")

# 기본 RAG 체인
rag_chain = (
    {
        "context": retriever | format_docs,
        "query": RunnablePassthrough()
    }
    | prompt | llm | StrOutputParser()
)

# 답변 + 출처 동시 반환 (RunnableParallel)
rag_with_sources = RunnableParallel(
    answer=rag_chain,
    sources=retriever | extract_sources
)

result = rag_with_sources.invoke("청약 자격은?")
print(result["answer"])
for src in result["sources"]:
    print(f"  - {src['id']} ({src['category']})")
```

#### RunnablePassthrough vs RunnableParallel

| 컴포넌트 | 역할 | 사용 예시 |
|----------|------|-----------|
| `RunnablePassthrough` | 입력 값을 그대로 통과 | `"question": RunnablePassthrough()` |
| `RunnableParallel` | 여러 체인 병렬 실행 → dict 반환 | `answer` + `sources` 동시 반환 |
| `RunnablePassthrough.assign` | 기존 dict에 새 키 추가 | 중간 결과 보존할 때 |

## 고려사항 & 질문 모음

### ❓ 임베딩 관련 질문

**Q. `cosine_similarity(embs[0], embs[1])` vs `cosine_similarity(embs[0], embs[1:])`의 차이는?**

- `cosine_similarity(A, B)`: A와 B의 모든 쌍에 대한 유사도 행렬 반환 (shape: `(|A|, |B|)`)
- `cosine_similarity([query_vec], doc_vecs)` → shape `(1, N)` → `[0]`으로 1D 변환 필요
- `cosine_similarity(embs[0], embs[1])` → 단일 벡터에 단일 벡터 → shape `(1,1)`, 스칼라 필요 시 `[0][0]`

**Q. `embed_query`와 `embed_documents`를 구분하는 이유는?**

- OpenAI의 text-embedding 모델은 내부적으로 query와 document를 다르게 최적화할 수 있음
- 검색 품질 향상을 위해 쿼리 임베딩과 문서 임베딩에 서로 다른 prefix를 붙이는 asymmetric 방식 사용

### ❓ RAG 아키텍처 관련 질문

**Q. RAG에서 키워드 검색과 벡터 검색의 차이는?**

| 항목 | 키워드 검색 (BM25) | 벡터 검색 (FAISS) |
|------|-------------------|-----------------|
| 방식 | 단어 일치 | 의미 유사도 |
| "아파트 청약 조건" 검색 시 | "청약", "조건" 단어 포함 문서 | 의미상 유사한 문서 |
| 동의어 처리 | ❌ 어려움 | ✅ 자연스럽게 처리 |
| 속도 | 빠름 | 상대적으로 느림 (벡터 연산) |

**Q. `similarity_score_threshold`에서 점수가 0~1 범위를 벗어나는 이유는?**

- FAISS는 기본적으로 L2 (유클리드) 거리를 사용
- LangChain이 내부적으로 L2 거리를 relevance score로 변환하는 과정에서 값이 범위를 벗어날 수 있음
- Cosine 기반 검색을 원한다면 `FAISS.from_documents` 전에 벡터를 정규화하거나 Chroma 사용 권장

**Q. `top_k` vs `top_p`의 차이는?**

- `top_k` (검색): 상위 k개 문서 반환 (retrieve 단계)
- `top_p` (LLM 생성): 누적 확률이 p 이상인 토큰만 샘플링 (generation 단계)
- RAG에서 `top_k`는 context 양을 조절, `top_p`는 LLM 출력 다양성을 조절

```python
def safe_ask(question, rag_chain):
    try:
        # 1. 타입 체크
        if not isinstance(question, str):
            raise ValueError("질문은 문자열이어야 합니다.")

        # 2. 공백 제거 후 빈 문자열 체크
        cleaned_question = question.strip()
        if not cleaned_question:
            raise ValueError("질문을 입력해주세요.")

        # 3. 길이 제한 (500자)
        if len(cleaned_question) > 500:
            raise ValueError(f"질문은 500자를 초과할 수 없습니다. (현재 {len(cleaned_question)}자)")

        # 4. 숫자만 입력된 경우 거부
        if cleaned_question.isdigit():
            raise ValueError("숫자만 입력된 질문은 처리할 수 없습니다.")

        return rag_chain.invoke(cleaned_question)

    except ValueError as e:
        return f"입력 오류: {str(e)}"
    except Exception as e:
        return f"처리 중 오류가 발생했습니다. {str(e)}"
```

```python
def search_faq(query: str, faq_data: dict, top_k: int = 3):
    # 1. 전체 FAQ 키워드 풀 수집
    extracted_keywords = set()
    for faq in faq_data:
        extracted_keywords.update(faq["keywords"])
    keywords_str = ", ".join(extracted_keywords)

    # 2. LLM으로 쿼리에서 관련 키워드 추출
    keyword_prompt = ChatPromptTemplate.from_messages([
        ("system", f"다음 키워드 목록에서 질문과 관련된 키워드를 {top_k}개 추출하세요: {keywords_str}"),
        ("user", "{question}")
    ])
    keyword_chain = keyword_prompt | llm | StrOutputParser()
    keyword_extracted = keyword_chain.invoke({"top_k": top_k, "question": query, ...})
    parsed_keywords = [kw.strip() for kw in keyword_extracted.split(",")]

    # 3. FAQ 키워드 매칭 점수 계산
    match_score = {}
    for faq in faq_data:
        match_score[faq["id"]] = 0
        for keyword in parsed_keywords:
            if keyword in faq["keywords"]:
                match_score[faq["id"]] += 1

    # 4. 최고 점수 FAQ 반환
    sorted_scores = sorted(match_score.items(), key=lambda item: item[1], reverse=True)
    best_id = sorted_scores[0][0]
    best_faq = next(faq for faq in faq_data if faq["id"] == best_id)

    return {"extracted_keywords": parsed_keywords, "best_score": sorted_scores[0][1], "best_faq": best_faq}
```

### 💻 핵심 코드 — Task 8 (FAQChatbotV2 클래스)

```python
import time
from langchain_community.vectorstores import FAISS
from langchain_core.runnables import RunnablePassthrough, RunnableParallel

class FAQChatbotV2:
    def __init__(self, faq_data, model_name="gpt-4o-mini"):
        self.llm = ChatOpenAI(model=model_name, temperature=0)
        self.embeddings = OpenAIEmbeddings(model="text-embedding-3-small")

        # 1. FAQ → Document 변환
        self.docs = []
        for sample in faq_data:
            content = f"Q: {sample['question']}\nA: {sample['answer']}"
            self.docs.append(Document(
                page_content=content,
                id=sample["id"],
                metadata={"category": sample["category"]}
            ))

        # 2. FAISS 벡터 스토어 구축
        self.vectorstore = FAISS.from_documents(self.docs, self.embeddings)

        # 3. Retriever (similarity, k=3)
        self.retriever = self.vectorstore.as_retriever(
            search_type="similarity",
            search_kwargs={"k": 3}
        )

        # 4. 프롬프트
        self.prompt = ChatPromptTemplate.from_messages([
            ("system", "다음 FAQ 문서를 참고하여 정확하게 답변하세요.\n\n{context}"),
            ("user", "{query}")
        ])

        # 5. RAG 체인
        self.rag_chain = (
            {
                "context": self.retriever | self.format_docs,
                "query": RunnablePassthrough()
            }
            | self.prompt | self.llm | StrOutputParser()
        )

        # 6. 답변 + 출처 동시 반환 체인
        self.chain_with_sources = RunnableParallel(
            answer=self.rag_chain,
            sources=self.retriever | self.extract_sources
        )

        print(f"FAQChatbotV2 초기화: {len(self.docs)}개 문서, {model_name}")

    def format_docs(self, docs):
        return "\n".join(doc.page_content for doc in docs)

    def extract_sources(self, docs):
        return [{"id": doc.id, "category": doc.metadata.get("category")} for doc in docs]

    def ask(self, question: str):
        start = time.time()
        result = self.chain_with_sources.invoke(question)
        elapsed = time.time() - start
        return {
            "answer": result["answer"],
            "sources": result["sources"],
            "time": round(elapsed, 2)
        }
```

### 💡 Task별 고려사항

#### Task 3 — FAISS 구축

- `FAISS.from_documents(docs, embeddings)`: 내부적으로 `embeddings.embed_documents()`를 호출하여 각 Document의 `page_content`를 벡터화
- `similarity_search_with_score()` 반환값은 **L2 거리** (낮을수록 유사) — Cosine과 방향 반대
- `allow_dangerous_deserialization=True`: `.pkl` 파일 역직렬화 시 보안 경고 우회 필요

#### Task 4 — searchType 비교

```python
# similarity — 단순 상위 k개 (빠름)
# mmr — 다양성 고려 (중복 응답 방지, fetch_k에서 k개 선택)
# similarity_score_threshold — 임계값 이상만 반환 (FAISS에서 불안정)
```

- **MMR (Maximal Marginal Relevance)**: `fetch_k`개를 먼저 검색한 후, 유사도와 다양성을 동시에 고려하여 `k`개 선택
- `lambda_mult=0`: 완전 다양성 우선, `lambda_mult=1`: 완전 유사도 우선 (기본값 0.5)

### 🔑 핵심 학습 포인트

1. **LLM은 도구**: 직접 답을 생성하는 것이 아니라, 검색 결과를 기반으로 답변 생성
2. **RAG의 핵심**: 좋은 검색 = 좋은 답변 → 검색 품질(임베딩 모델, 청크 전략, top_k)이 전체 품질 결정
3. **LCEL의 힘**: 복잡한 파이프라인을 `|` 연산자로 선언적으로 구성
4. **Document 객체**: LangChain 생태계의 공통 데이터 단위, `page_content` + `metadata`
5. **벡터 DB 선택**: 프로토타입 → FAISS, 프로덕션 → Qdrant/Pinecone
