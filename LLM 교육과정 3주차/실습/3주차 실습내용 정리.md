### 3-1. 프롬프트 엔지니어링 기초

#### 좋은 프롬프트의 4요소

프롬프트 엔지니어링은 LLM 파라미터를 수정하지 않고 **입력 텍스트 설계만으로** 출력 품질을 제어하는 기술 분야.

> 📄 참고 자료:
> - OpenAI Prompt Engineering Guide: https://developers.openai.com/api/docs/guides/prompt-engineering
> - 프롬프트 엔지니어링 최신 연구 (2024): https://arxiv.org/pdf/2408.02442
> - 한국어 프롬프팅 연구: https://arxiv.org/pdf/2311.10054
> - Prompt Formatting 영향 분석: https://arxiv.org/pdf/2411.10541
> - ExpertPrompting: https://arxiv.org/pdf/2305.14688

| 요소 | 설명 | 예시 |
|------|------|------|
| **지시 (Instruction)** | 수행할 작업 명시 | `"다음 텍스트의 감정을 분석하세요"` |
| **컨텍스트 (Context)** | 배경 정보 제공 | `"고객 리뷰 분석 시스템입니다"` |
| **입력 (Input)** | 처리할 실제 데이터 | 리뷰 텍스트 원문 |
| **출력 형식 (Output Format)** | 원하는 응답 구조 지정 | `"감정: [긍정/부정/혼합], 이유: [한 줄]"` |

#### `build_prompt()` 함수

```python
def build_prompt(instructions, context="", input_data="", output_format=""):
    parts = []
    parts.append(f"[지시] {instructions}")
    if context:
        parts.append(f"[컨텍스트] {context}")
    if input_data:
        parts.append(f"[입력] {input_data}")
    if output_format:
        parts.append(f"[출력 형식] {output_format}")
    return '\n'.join(parts)

# 사용 예시
prompt = build_prompt(
    instructions='다음 텍스트의 감정을 분석하세요',
    context='고객 리뷰 분석 시스템입니다.',
    input_data='배송은 빨랐는데 포장이 엉망입니다.',
    output_format='감정: [긍정/부정/혼합], 이유: [한 줄 설명]'
)
# 출력:
# [지시] 다음 텍스트의 감정을 분석하세요
# [컨텍스트] 고객 리뷰 분석 시스템입니다.
# [입력] 배송은 빨랐는데 포장이 엉망입니다.
# [출력 형식] 감정: [긍정/부정/혼합], 이유: [한 줄 설명]

result = llm.invoke([HumanMessage(content=prompt)])
# → "감정: 혼합, 이유: 빠른 배송 긍정 + 포장 불량 부정"
```

### 3-2. 역할 기반 어시스턴트 & CoT

#### `RoleBasedAssistant` 클래스

서로 다른 역할(CEO, CTO, CFO 등)을 가진 어시스턴트를 만들어 **다양한 관점의 의견을 수집**하는 패턴.

> 📄 참고 논문: Zero-Shot Reasoning with Role-Play Prompting (2023): https://arxiv.org/pdf/2308.07702

```python
class RoleBasedAssistant:
    def __init__(self, role_name, system_prompt):
        self.role_name = role_name
        self.system_prompt = system_prompt

    def ask(self, question):
        messages = [
            SystemMessage(content=self.system_prompt),
            HumanMessage(content=question)
        ]
        return llm.invoke(messages).content

# 전문가 패널 구성
experts = [
    RoleBasedAssistant(
        role_name='CEO',
        system_prompt='당신은 기업의 CEO입니다. 비즈니스 비전, 시장 경쟁력, 전사적 전략 관점에서 의견을 3문장으로 제시해주세요.'
    ),
    RoleBasedAssistant(
        role_name='CTO',
        system_prompt='당신은 기업의 CTO입니다. 기술적 타당성, 인프라 구축, 데이터 보안 관점에서 의견을 3문장으로 제시해주세요.'
    ),
    RoleBasedAssistant(
        role_name='CFO',
        system_prompt='당신은 기업의 CFO입니다. ROI, 예산 할당, 비용 대비 편익, 재무적 리스크 관점에서 의견을 3문장으로 제시해주세요.'
    ),
]

# 각 전문가 의견 수집
question = 'AI 도입 전략에 대한 당신의 의견을 들려주세요.'
opinions = []
for expert in experts:
    answer = expert.ask(question)
    opinions.append(f"{expert.role_name}: {answer}")

# 사회자가 종합 정리
moderator = RoleBasedAssistant(
    role_name='사회자',
    system_prompt='당신은 경영 회의 사회자입니다. 여러 전문가 의견을 종합해 합의점을 3문장으로 도출해주세요.'
)
summary_input = "다음 의견을 종합 정리하세요:\n\n" + "\n\n".join(opinions)
moderator.ask(summary_input)
```

#### CoT (Chain of Thought)

```python
# Zero-shot CoT: "단계별로 생각해보세요" 추가만으로 성능 향상
task_data = "2026년 우리 회사 매출: Q1: 50억, Q2: 65억, Q3: 45억, Q4: 80억"

# ❌ 단순 프롬프트
single_prompt = f"다음 데이터를 분석하고 인사이트를 제공하세요: {task_data}"

# ✅ CoT 프롬프트
cot_prompt = f"""
다음 데이터를 단계별로 분석하세요.
1단계: 수치 계산 (총합, 성장률)
2단계: 패턴 파악 (증감 추세)
3단계: 인사이트 도출 (원인과 개선점)

데이터: {task_data}
"""
```

> 💡 **LLM에서 'Shot'의 의미**: 예시(Example)의 수를 의미.
> - **Zero-shot**: 예시 없이 바로 질문 (= GPT에 그냥 요청하는 것)
> - **Few-shot**: 예시를 조금 제공 후 질문
> - Vision Task의 Zero-shot(학습 없이 분류)과는 다른 개념

### 3-3. 출력 형식 제어 (Output Formatting)

#### 프롬프트 포맷 3가지 비교

동일한 작업을 요청해도 프롬프트 형식에 따라 JSON 파싱 편의성이 달라짐.

| 방식 | 특징 | JSON 파싱 결과 |
|------|------|---------------|
| **Plain 텍스트** | 자연어로 요청 | 마크다운 코드블록 래퍼(` ```json `) 포함 가능 → `json.loads()` 전 전처리 필요 |
| **Markdown 구조** | `##`, `-`, `**` 등 마크다운으로 구조화 | 순수 JSON만 출력되는 경향 (더 깔끔) |
| **JSON 예시 포함** | 출력 구조를 직접 보여줌 | 마크다운 래퍼 포함 가능 |

```python
# Plain 텍스트 방식 (마크다운 래퍼 포함될 수 있음)
plain_prompt = """다음 제품 리뷰를 분석해줘, 전체 감정(긍정/부정/혼합), 1-5점 점수, 장점 목록
단점 목록, 핵심 키워드 3개를 알려줘, 리뷰: "이 노트북 정말 가벼워서 좋아요! 하지만 키보드 타건감이 아쉽네요."
유효한 json만 출력하세요"""

# Markdown 구조 방식 (순수 JSON만 출력되는 경향)
markdown_prompt = """# 제품 리뷰 감정 분석
## 입력 리뷰
> "이 노트북 정말 가벼워서 좋아요! 하지만 키보드 타건감이 아쉽네요."

## 분석 항목
- **overall_sentiment**: 긍정/부정/혼합 중 하나
- **score**: 1-5 점수
- **pros**: 장점 리스트
- **cons**: 단점 리스트
- **keywords**: 핵심 키워드 3개

## 출력 형식
유효한 json만 출력하세요. 다른 텍스트를 포함하지 마세요.
"""

print(llm.invoke([HumanMessage(content=plain_prompt)]).content)
print(llm.invoke([HumanMessage(content=markdown_prompt)]).content)
```

> 💡 **포맷 제한과 성능**: JSON 등 형식 제한 시 창의적 답변 품질이 낮아질 수 있으나, 오히려 성능이 올라가는 케이스도 존재. 참고: HuggingFace - Evaluation of Structured Outputs (https://huggingface.co/blog/evaluation-structured-outputs)

#### 제약 조건 설계 원칙

```python
# ❌ 부정 제약 (효과 낮음)
"거짓말로 지어내지 마세요"
```

### 3-4. ConstrainedPrompt & 메서드 체이닝

#### 메서드 체이닝 핵심 원리

여러 메서드를 점(`.`)으로 이어서 한 번에 실행하는 방식. 각 메서드가 `return self`를 반환해야 체이닝이 가능함.

#### `ConstrainedPrompt` 클래스

```python
class ConstrainedPrompt:
    def __init__(self, base_instruction):
        self.instruction = base_instruction
        self.constraints = []

    def add_length(self, description):
        self.constraints.append(f"[길이] {description}")
        return self  # ← return self가 메서드 체이닝의 핵심

    def add_content(self, description):
        self.constraints.append(f"[내용] {description}")
        return self

    def add_format(self, description):
        self.constraints.append(f"[형식] {description}")
        return self

    def add_style(self, description):
        self.constraints.append(f"[스타일] {description}")
        return self

    def build(self):
        parts = [self.instruction, "\n제약조건:"]
        for c in self.constraints:
            parts.append(f" - {c}")
        return '\n'.join(parts)

    def execute(self, **kwargs):
        # **kwargs: 정의되지 않은 임의 키워드 인자 수용 (temperature 등)
        prompt = self.build()
        return llm.invoke([HumanMessage(content=prompt)]).content

# 사용 예시 (메서드 체이닝)
result = (
    ConstrainedPrompt('클라우드 컴퓨팅의 장점을 설명해주세요')
    .add_length("5개의 불릿포인트")
    .add_content('비용, 확장성, 보안 관점을 반드시 포함')
    .add_format('각 포인트는 한 줄로, 이모지로 시작')
    .add_style('IT 비전공 경영진 대상, 전문 용어에 괄호로 설명 추가')
    .execute()
)

# 실습: 광고 카피 생성
product = "에어프로 맥스 무선 헤드폰"
features = ["40시간 배터리", "멀티포인트 연결", "30dB 노이즈캔슬링", "300g 경량 설계"]

answer = (
    ConstrainedPrompt(f"{product}의 장점을 설명해주세요. 특징: {'. '.join(features)}")
    .add_length("5개의 불릿포인트")
    .add_content("핵심 특징 항목에 대한 설명을 포함")
    .add_format('각 포인트는 한 줄로, 이모지로 시작')
    .add_style('대중 및 주요 소비자 대상, 장점을 극대화')
    .execute()
)
```

> ⚠️ **`**kwargs` 주의사항**: `execute()`에서 `**kwargs`로 받은 `temperature=0.3` 같은 인자는 현재 코드 내에서 `llm.invoke()`에 전달되지 않음. 실제 적용하려면 `llm = ChatOpenAI(..., temperature=kwargs.get('temperature', 0.7))` 처럼 명시 필요.

### 3-5. RAG 컨텍스트 주입 & 위치 효과

> 📄 RAG 참고 논문 — Precise Zero-Shot Dense Retrieval without Relevance Labels (2022): https://arxiv.org/pdf/2211.01910

#### 컨텍스트 주입 기본 패턴

```python
company_policy = """
[모두컴퍼니 재택근무 정책 v2.3]
- 주 3일 재택, 2일 출근 (화/목 필수 출근)
- 재택근무 시 오전 9시까지 Slack 상태 '업무중' 설정 필수
- 해외 원격근무는 최대 연속 2주까지 가능 (사전 승인 필요)
- 야간근무(22시 이후) 시 익일 오후 출근 가능
- 재택근무 장비 지원금: 연 100만원 (영수증 제출)
"""

messages = [
    SystemMessage(content="회사 정책 안내 챗봇입니다. 주어진 정책 문서 내에서만 답변하세요."),
```

### 3-6. Few-Shot 프롬프팅

#### 핵심 클래스 4종

| 클래스 | 용도 | 특징 |
|--------|------|------|
| `PromptTemplate` | 단일 텍스트 프롬프트 | `input_variables` + `template` |
| `FewShotPromptTemplate` | 텍스트 기반 퓨샷 | `prefix` + 예시들 + `suffix` 조합 |
| `ChatPromptTemplate` | 채팅 형식 프롬프트 | `from_messages([('role', 'content')])` |
| `FewShotChatMessagePromptTemplate` | 채팅 형식 퓨샷 | Human/AI 쌍으로 예시 구성 |

#### `FewShotPromptTemplate` 사용법

```python
from langchain_core.prompts import (
    FewShotPromptTemplate, PromptTemplate,
    ChatPromptTemplate, FewShotChatMessagePromptTemplate
)

# 1. 예시 데이터 준비
examples = [
    {"input": "이 제품 정말 최고입니다!", "output": "긍정"},
    {"input": "이 제품 품질이 너무 안좋아요", "output": "부정"},
    {"input": "보통이에요. 무난합니다.", "output": "중립"},
    {"input": "디자인은 좋은데 성능이 아쉬워요", "output": "혼합"},
]

# 2. 예시 하나를 어떻게 표시할지 정의 (PromptTemplate)
example_template = PromptTemplate(
    input_variables=['input', 'output'],
    template="리뷰: {input}\n감정: {output}"
)

# 3. FewShotPromptTemplate 구성
fewshot_prompt = FewShotPromptTemplate(
    examples=examples,           # 예시 데이터 리스트
    example_prompt=example_template,  # 예시 하나의 렌더링 방식
    prefix='다음 예시를 참고해서 리뷰의 감정을 분류해주세요\n',  # 예시 전 지시문
    suffix="\n리뷰: {input}\n감정:",   # 예시 후 실제 입력
    input_variables=['input'],         # 외부에서 받을 변수
    example_separator="\n\n"           # 예시 간 구분자
)

# 4. 완성된 프롬프트 확인
formatted = fewshot_prompt.format(input='가격은 비싸지만 품질이 뛰어나요')
print(formatted)
# 다음 예시를 참고해서 리뷰의 감정을 분류해주세요
#
# 리뷰: 이 제품 정말 최고입니다!
# 감정: 긍정
# ...
# 리뷰: 가격은 비싸지만 품질이 뛰어나요
# 감정:
```

#### Zero-shot vs Few-shot 성능 비교

```python
# Zero-shot 평가 함수
def evaluate_zero_shot(dataset):
    results = []
    for item in dataset:
        response = llm.invoke([
            SystemMessage(content="리뷰의 감정을 분류하세요. 반드시 '긍정','부정','중립','혼합' 중 하나만 출력하세요."),
            HumanMessage(content=item['text'])
        ]).content
        predicted = response.strip()
        results.append({
            'text': item['text'],
            'expected': item['expected'],
            'predicted': predicted,
            'correct': predicted == item['expected'],
        })
    return results

# Few-shot 평가 함수
def evaluate_few_shot(dataset, examples):
    results = []
    # System 메시지 + 예시(Human/AI 쌍) 미리 구성
    base_messages = [SystemMessage(content="리뷰의 감정을 분류하세요. 반드시 '긍정','부정','중립','혼합' 중 하나만 출력하세요.")]
    for ex in examples:
        base_messages.append(HumanMessage(content=ex['input']))
        base_messages.append(AIMessage(content=ex['output']))

    for item in dataset:
        messages = base_messages + [HumanMessage(content=item['text'])]
        response = llm.invoke(messages).content
        predicted = response.strip()
        results.append({
            'text': item['text'],
            'expected': item['expected'],
            'predicted': predicted,
            'correct': predicted == item['expected'],
        })
    return results

# 실행 및 결과 확인
fewshot_results = evaluate_few_shot(eval_dataset, few_shot_examples)
pd.DataFrame(fewshot_results)
# → 8개 테스트 케이스 8/8 (100%) 정확도 달성
```

> 💡 **Few-shot 예시 설계 원칙** (예시가 성능의 70%를 결정한다는 연구 결과도 존재):
> - **다양성**: 편향(bias) 방지를 위해 다양한 케이스 포함
> - **균형**: 긍정/부정 등 클래스 분포가 균등해야 함
> - **관련성**: 실제 사용 케이스와 유사한 예시 선택

> 📄 관련 논문:
> - Chain of Thought Prompting 원본 (2022): https://arxiv.org/pdf/2205.11916
> - 예시 순서에 따른 성능 차이: https://arxiv.org/pdf/2104.08786
> - Few-shot 편향 분석: https://arxiv.org/pdf/2102.09690
> - KNN 기반 예시 선택: https://arxiv.org/pdf/2101.06804
> - 라벨 무작위 부여 시 성능 변화 없음 (포맷 학습 가설): https://arxiv.org/pdf/2202.12837
> - Automatic Example Optimization (APE): https://arxiv.org/pdf/2209.01975

예시 풀이 클 때 **입력과 의미적으로 가장 유사한** 예시 k개를 자동 선택.

#### `SemanticSimilarityExampleSelector` 사용법

```python
from langchain_core.example_selectors import SemanticSimilarityExampleSelector
from langchain_core.vectorstores import InMemoryVectorStore
from langchain_openai import OpenAIEmbeddings

large_examples = [
    {"input": "정말 만족합니다. 강력 추천!", "output": "긍정"},
    {"input": "배송이 빠르고 포장이 꼼꼼해요.", "output": "긍정"},
    {"input": "품질이 최악입니다. 환불 요청했어요.", "output": "부정"},
    {"input": "고객센터 응대가 너무 불친절합니다.", "output": "부정"},
    {"input": "보통이에요. 특별한 점은 없습니다.", "output": "중립"},
    {"input": "디자인은 예쁜데 내구성이 약해요.", "output": "혼합"},
    # ...
]

example_selector = SemanticSimilarityExampleSelector.from_examples(
    large_examples,                                      # 전체 예시 풀
    OpenAIEmbeddings(model="text-embedding-3-small"),   # 임베딩 모델
    InMemoryVectorStore,                                 # 벡터 저장소 (휘발성)
    k=3                                                  # 선택할 예시 수
)

# 입력에 따라 자동으로 유사한 예시 3개 선택
queries = ['배송이 너무 느려요', '값은 좀 나가지만 성능은 훌륭해요', '무난한 제품이에요']
for q in queries:
    selected = example_selector.select_examples({'input': q})
    print(f"query: {q}")
    for s in selected:
        print(f"  selected: {s['output']}, {s['input']}")

# 출력 예시:
# query: 배송이 너무 느려요
#   selected: 긍정, 배송이 빠르고 포장이 꼼꼼해요.
#   selected: 부정, 고객센터 응대가 너무 불친절합니다.
#   selected: 부정, 택배가 분실되었어요.
```

### 3-8. 대화 메모리 관리 기초

#### LLM의 무상태(Stateless) 특성

LLM은 자체 기억이 없어서 매 요청마다 **전체 대화 히스토리를 직접 전달**해야 함.

```python
# 기본 방식: 메시지 리스트에 수동으로 append
messages = [SystemMessage(content='당신은 친절한 AI 어시스턴트입니다.')]

# 1번 대화
messages.append(HumanMessage(content='내 이름은 abc입니다.'))
response1 = llm.invoke(messages)
print(response1.content)  # "반가워요, abc님!"

# AI 응답도 반드시 추가해야 다음 대화에서 기억함
messages.append(AIMessage(content=response1.content))
messages.append(HumanMessage(content='내 이름이 뭔가요?'))
response2 = llm.invoke(messages)
print(response2.content)  # "abc님, 당신의 이름은 'abc'입니다!"
```

#### `tiktoken` 토큰 카운팅

대화가 길어질수록 토큰 비용이 증가하므로 모니터링 필요.

```python
import tiktoken

enc = tiktoken.encoding_for_model('gpt-4o-mini')

def count_tokens(messages):
    total = 0
    for msg in messages:
        total += len(enc.encode(msg.content))
        total += 4  # 메시지당 구조 오버헤드
    return total

# 대화 턴이 쌓일수록 토큰 증가 확인
conversation = [SystemMessage(content='당신은 친절한 AI 어시스턴트입니다.')]
sample_exchanges = [
    ('내 이름은 abc 입니다.', '반가워요, abc님!'),
    ('내 이름은 뭔가요?', 'abc님, 이름은 abc입니다!')
]
for i, (user_msg, ai_msg) in enumerate(sample_exchanges):
    conversation.append(HumanMessage(user_msg))
    conversation.append(AIMessage(ai_msg))
    print(f"턴 {i+1} | 누적 토큰: {count_tokens(conversation)}")
# 턴 0 | 누적 토큰: 47
# 턴 1 | 누적 토큰: 83
```

### 3-9. 메모리 관리 전략 3종

#### 전략 비교

| 전략 | 클래스 | 장점 | 단점 |
|------|--------|------|------|
| **전체 저장** | `ConversationBufferMemory` | 완전한 대화 히스토리 유지 | 대화가 길어질수록 토큰 폭증 |
| **윈도우 슬라이딩** | `trim_messages` / 수동 슬라이싱 | 토큰 수 일정하게 유지 | 오래된 정보 소멸 (이름 등 잊음) |
| **요약 저장** | `ConversationSummaryMemory` | 긴 대화도 압축 유지 | 세부 디테일 손실, LLM 추가 호출 비용 |

#### 1. `ConversationBufferMemory`

```python
from langchain_classic.memory import ConversationBufferMemory

# return_messages=True: Message 객체 리스트 반환 (LLM에 직접 전달 가능)
# return_messages=False: 텍스트 문자열 반환 (로깅, 디버깅용)
memory = ConversationBufferMemory(return_messages=True)

memory.save_context(
    {'input': '안녕하세요. 저는 abc입니다.'},
    {'output': '안녕하세요 abc님! 만나서 반갑습니다.'}
)
memory.save_context(
    {'input': '오늘 날씨가 좋네요!'},
    {'output': '네, 정말 화창한 날씨입니다.'}
)

history = memory.load_memory_variables({})
# {'history': [HumanMessage(...), AIMessage(...), HumanMessage(...), AIMessage(...)]}

for msg in history['history']:
    print(msg.content)
```

#### 2. `trim_messages` & 슬라이딩 윈도우

```python
from langchain_core.messages import trim_messages

# trim_messages: 최근 N개 메시지만 유지
trimmer = trim_messages(
    max_tokens=4,          # token_counter=len이면 메시지 개수 기준
    strategy='last',       # 'last': 최신 메시지 기준으로 자름
    token_counter=len,     # 토큰 카운팅 함수 (len → 메시지 수 카운트)
    start_on='human'       # Human 메시지로 시작하도록 보장
)
trimmed = trimmer.invoke(messages)
# 6개 메시지(3턴) → 4개(2턴)로 축소

# 수동 슬라이딩 윈도우 (간단한 구현)
messages = []
K = 2  # 유지할 최근 턴 수

def add_turn(user_input, ai_output):
    global messages
    messages.append(HumanMessage(content=user_input))
    messages.append(AIMessage(content=ai_output))
    messages = messages[-(K * 2):]  # 최근 K턴(K*2개 메시지)만 유지

add_turn('첫번째, 내 이름은 abc입니다.', '안녕하세요, abc님')
# messages: [HumanMessage, AIMessage]   (1턴)
add_turn('두번째, 나는 학생입니다.', '학생이군요 멋지십니다.')
# messages: [HumanMessage, AIMessage, HumanMessage, AIMessage]   (2턴 = 최대)
add_turn('세번째, 나는 파이썬 좋아함.', '파이썬 좋아하는군요')
# messages: [HumanMessage(두번째), AIMessage(두번째), HumanMessage(세번째), AIMessage(세번째)]
# → 첫 번째 대화 소멸!
```

#### 3. `ConversationSummaryMemory`

```python
from langchain_classic.memory import ConversationSummaryMemory

# 대화를 요약해서 저장 → 긴 대화도 압축 유지
# 요약 작업 자체에 LLM이 필요 (별도 LLM 인스턴스 권장)
summary_llm = ChatOpenAI(model='gpt-4o-mini')
summary_memory = ConversationSummaryMemory(
    llm=summary_llm,
    return_messages=False  # 텍스트 요약본으로 반환
)

conversations = [
    ('첫번째, 내 이름은 abc입니다.', '안녕하세요, abc님'),
    ('두번째, 나는 학생', '학생이군요'),
    ('세번째, 나는 파이썬 좋아함.', '파이썬 좋아하는군요')
]

# 대화가 추가될 때마다 LLM이 자동으로 요약문 업데이트 (누적 덮어쓰기)
for user_msg, ai_msg in conversations:
    summary_memory.save_context({'input': user_msg}, {'output': ai_msg})

result = summary_memory.load_memory_variables({})
# {'history': 'abc라는 사람이 자신을 소개하며 학생이고 파이썬을 좋아한다고 밝혔다.'}
```

> 💡 **LangChain 메모리의 미래**: 현재 `ConversationBufferMemory` 같은 레거시 메모리 클래스들은 **LangGraph**의 State 기반 설계로 대체되는 방향으로 발전 중. LangGraph는 모든 대화 컨텍스트와 체인 상태를 하나의 State 객체로 통합 관리함.

#### 보충: `ConversationBufferWindowMemory`

`trim_messages`를 수동으로 구현하는 대신, LangChain 내장 클래스 사용 가능.

```python
from langchain_classic.memory import ConversationBufferWindowMemory

# k: 유지할 최근 대화 턴 수
window_memory = ConversationBufferWindowMemory(k=2, return_messages=True)
```

#### 보충: 메시지 언패킹 (`*` 연산자)

```python
# chat_history.messages는 리스트 → *로 언패킹하면 개별 요소로 풀림
messages = [
    *chat_history.messages,  # 리스트를 개별 Message 객체로 풀어서 삽입
    HumanMessage(content='내 이름이 뭔가요?')
]
# 결과: [HumanMessage(...), AIMessage(...), ..., HumanMessage('내 이름이 뭔가요?')]
# → flat한 메시지 리스트가 되어 llm.invoke()에 바로 전달 가능
```

#### 보충: `RunnableWithMessageHistory` & `MessagesPlaceholder`

세션 기반 대화 히스토리를 Runnable 체인에 자동 주입하는 LangChain 핵심 패턴.

```python
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain_core.runnables.history import RunnableWithMessageHistory

# 1. 프롬프트에 히스토리 삽입 위치 지정
prompt = ChatPromptTemplate.from_messages([
    ('system', '당신은 친절한 AI 어시스턴트입니다.'),
    MessagesPlaceholder(variable_name='history'),  # ← 대화 히스토리가 여기에 삽입됨
    ('human', '{input}')
])
chain = prompt | llm

# 2. 세션별 히스토리 저장소
store = {}
def get_session_history(session_id):
    if session_id not in store:
        store[session_id] = InMemoryChatMessageHistory()
    return store[session_id]

# 3. RunnableWithMessageHistory로 체인 래핑
chain_with_history = RunnableWithMessageHistory(
    chain,                              # 대화 로직이 담긴 체인
    get_session_history,                # 세션 ID → 히스토리 객체 반환 함수
    input_messages_key='input',         # 사용자 입력 키
    history_messages_key='history'      # 프롬프트 내 히스토리 플레이스홀더 키
)

# 4. 세션 ID를 config로 전달하여 실행
config = {'configurable': {'session_id': 'user_001'}}
r1 = chain_with_history.invoke({'input': '안녕하세요, 제 이름은 ABC입니다.'}, config=config)
r2 = chain_with_history.invoke({'input': '제 이름이 뭔가요?'}, config=config)
# → 같은 session_id이므로 이전 대화를 기억하고 "ABC"라고 답변
```

#### 보충: 커스텀 `SummaryMemory` & `SummaryChatbot`

> ⚠️ LangChain 공식 `ConversationSummaryMemory`는 내부 동작이 기대와 다를 수 있어, 실습에서는 직접 커스텀 클래스를 구현함.

```python
class SummaryMemory:
    def __init__(self, summary_interval=3):
        self.summary = ''
        self.recent_messages = []
        self.summary_interval = summary_interval
        self.turn_count = 0

    def _summarize(self, text):
        response = llm.invoke([
            SystemMessage(content='주어진 대화 내용을 간결하게 한국어로 요약하세요.'),
            HumanMessage(content=f'이전 요약:\n{self.summary}\n\n새 대화:\n{text}')
        ])
        return response.content

    def add_exchange(self, user_msg, ai_msg):
        self.recent_messages.append(f'사용자: {user_msg}')
        self.recent_messages.append(f'AI: {ai_msg}')
        self.turn_count += 1
        # summary_interval 턴마다 자동 요약
        if self.turn_count % self.summary_interval == 0:
            conversation_text = '\n'.join(self.recent_messages)
            self.summary = self._summarize(conversation_text)
            self.recent_messages = []

    def get_context(self):
        parts = []
        if self.summary:
            parts.append(f'[이전 대화 요약] {self.summary}')
        if self.recent_messages:
            parts.append(f'[최근 대화]\n' + '\n'.join(self.recent_messages))
        return '\n\n'.join(parts)


class SummaryChatbot:
    def __init__(self, system_prompt='당신은 도움이 되는 AI 어시스턴트입니다.', summary_interval=3):
        self.system_prompt = system_prompt
        self.memory = SummaryMemory(summary_interval=summary_interval)

    def chat(self, user_input):
        context = self.memory.get_context()
        messages = [SystemMessage(content=self.system_prompt)]
        if context:
            messages.append(SystemMessage(content=f'대화 맥락:\n{context}'))
        messages.append(HumanMessage(content=user_input))
        response = llm.invoke(messages)
        ai_response = response.content
        self.memory.add_exchange(user_input, ai_response)
        return ai_response

# 사용 예시
bot = SummaryChatbot(system_prompt='당신은 IT 커머스 상담원입니다.', summary_interval=2)
bot.chat('노트북을 사고 싶은데, 어떤 제품이 있나요?')
```

### ❓ Few-Shot 관련

**Q. Few-shot에서 라벨을 무작위로 줘도 성능 변화가 없다는 연구는 무슨 의미인가요?**

- 논문 (2022, https://arxiv.org/pdf/2202.12837)에 따르면, 예시의 **라벨이 틀려도** 성능이 크게 떨어지지 않음.
- 이는 모델이 예시로부터 학습하는 것이 **정답 라벨**보다 **출력 포맷과 입출력 패턴** 위주임을 시사.
- 즉, Few-shot은 "어떤 형식으로 답해야 하는지"를 가르치는 효과가 큼.

**Q. `SemanticSimilarityExampleSelector`와 직접 예시 리스트를 사용하는 것의 차이는?**

| 항목 | 고정 예시 리스트 | SemanticSimilarityExampleSelector |
|------|-----------------|----------------------------------|
| 예시 선택 | 항상 동일한 예시 | 입력마다 다른 예시 동적 선택 |
| 편향 위험 | 높음 (특정 케이스 편중 가능) | 낮음 (입력에 맞는 예시 선택) |
| 구현 비용 | 낮음 | OpenAI Embeddings API 호출 필요 |
| 적합한 경우 | 예시 수가 적을 때 | 예시 풀이 크고 다양할 때 |
