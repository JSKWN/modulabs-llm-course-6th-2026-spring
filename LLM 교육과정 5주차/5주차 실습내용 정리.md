## 핵심 개념 상세 정리

### 5-1. 벡터 검색과 재순위화 기법 (Reranking)

기본 벡터 검색은 임베딩 유사도만 사용 → 정밀도에 한계가 있음.
재순위화는 초기 검색 결과를 더 정교한 기준으로 다시 정렬하는 과정.

#### Bi-encoder vs Cross-encoder

| 방식 | 구조 | 속도 | 정밀도 | 활용 |
|---|---|---|---|---|
| Bi-encoder | 쿼리·문서 독립 인코딩 | 빠름 | 보통 | 초기 검색 (FAISS 등) |
| Cross-encoder | 쿼리+문서 동시 인코딩 | 느림 | 높음 | 재순위화 단계 |

> 실무: Bi-encoder로 빠르게 후보를 좁히고 → Cross-encoder/LLM으로 재정렬

#### vector_search — FAISS 유사도 검색

```python
def vector_search(query, vectorstore, top_k=5):
    # FAISS에서 유사도 검색 수행 → (Document, 거리) 튜플 반환
    results = vectorstore.similarity_search_with_score(query, k=top_k)
    # 거리(distance)를 유사도(similarity)로 변환: 1/(1+거리)
    return [(doc, 1.0 / (1.0 + score)) for doc, score in results]
```

#### keyword_rerank — 키워드 매칭 기반 재순위화

쿼리 단어가 문서에 몇 번 등장하는지 세어 가중치를 부여.

```python
def keyword_rerank(query, search_results):
    query_terms = set(query.lower().split())  # 쿼리를 단어 집합으로 변환
    reranked = []
    for doc, orig_score in search_results:
        doc_terms = doc.page_content.lower().split()
        keyword_hits = sum(1 for t in doc_terms if t in query_terms)  # 매칭 횟수
        new_score = orig_score + 0.1 * keyword_hits  # 기존 점수 + 키워드 가중치
        reranked.append((doc, new_score, orig_score))
    reranked.sort(key=lambda x: x[1], reverse=True)
    return reranked
```

#### BM25Reranker — TF-IDF 개선 키워드 랭킹

| 파라미터 | 기본값 | 역할 |
|---|---|---|
| `k1` | 1.5 | 단어 빈도(TF) 포화 계수. 높을수록 빈도 영향 증가 |
| `b` | 0.75 | 문서 길이 정규화 계수. 1에 가까울수록 길이 보정 강함 |

```python
class BM25Reranker:
    def __init__(self, k1=1.5, b=0.75):
        self.k1 = k1
        self.b = b

    def rerank(self, query, search_results):
        docs = [doc for doc, _ in search_results]
        tokenized = [doc.page_content.lower().split() for doc in docs]
        avg_dl = np.mean([len(t) for t in tokenized])  # 평균 문서 길이

        # DF(문서 빈도) 계산: 각 단어가 몇 개의 문서에 등장하는지
        df_count = Counter()
        for tokens in tokenized:
            for t in set(tokens):
                df_count[t] += 1
        N = len(docs)

        query_tokens = query.lower().split()
        scored = []
        for i, doc in enumerate(docs):
            score = 0.0
            doc_len = len(tokenized[i])
            tf_count = Counter(tokenized[i])  # 문서 내 단어별 빈도
            for qt in query_tokens:
                tf = tf_count.get(qt, 0)
                if tf == 0:
                    continue
                idf = math.log((N - df_count.get(qt,0) + 0.5) / (df_count.get(qt,0) + 0.5) + 1)
                numerator = tf * (self.k1 + 1)
                denominator = tf + self.k1 * (1 - self.b + self.b * doc_len / avg_dl)
                score += idf * numerator / denominator
            scored.append((doc, score))
        scored.sort(key=lambda x: x[1], reverse=True)
        return scored
```

#### hybrid_rerank — BM25 + LLM 하이브리드

```
최종 점수 = bm25_weight × BM25_norm + (1 - bm25_weight) × LLM_norm
```

- BM25와 LLM 점수의 스케일이 다르므로 **min-max 정규화** 후 결합
- `bm25_weight` 기본값 0.3 → LLM 점수를 더 신뢰

```python
def hybrid_rerank(query, search_results, bm25_weight=0.3):
    bm25 = BM25Reranker()
    bm25_scored = bm25.rerank(query, search_results)
    bm25_map = {doc.metadata['id']: score for doc, score in bm25_scored}

    llm_scored = llm_rerank(query, search_results, top_k=len(search_results))
    llm_map = {doc.metadata['id']: score for doc, score, _ in llm_scored}

    # min-max 정규화: 두 스코어의 스케일을 0~1로 맞춤
    def normalize(scores_dict):
        vals = list(scores_dict.values())
        min_, max_ = min(vals), max(vals)
        range_ = max_ - min_ if max_ > min_ else 1e-8
        return {k: (v - min_) / range_ for k, v in scores_dict.items()}

    bm25_norm = normalize(bm25_map)
    llm_norm = normalize(llm_map)

    combined = []
    for doc, _ in search_results:
        did = doc.metadata['id']
        score = bm25_weight * bm25_norm.get(did, 0) + (1 - bm25_weight) * llm_norm.get(did, 0)
        combined.append((doc, score))
    combined.sort(key=lambda x: x[1], reverse=True)
    return combined
```

### 5-2. 스코어 필터링 (Score Filtering)

재순위화 이후 관련성이 낮은 문서를 제거 → 컨텍스트 품질 향상.
LLM 컨텍스트 윈도우를 효율적으로 활용하기 위해 필요.

#### ScoreFilter — 3가지 전략

| 전략 | 방식 | 특징 |
|---|---|---|
| Fixed Threshold | 고정 점수(0.5) 이상만 유지 | 단순, 도메인별 임계값 설정 필요 |
| Dynamic Threshold | `평균 - std_factor × 표준편차` 이상 | 점수 분포에 따라 자동 조정 |
| Score Gap | 연속 문서 간 점수 차이가 가장 큰 지점에서 절단 | 자연스러운 품질 경계 탐지 |

```python
class ScoreFilter:
    @staticmethod
    def fixed_threshold(scored_docs, threshold=0.5):
        return [(doc, s) for doc, s in scored_docs if s >= threshold]

    @staticmethod
    def dynamic_threshold(scored_docs, std_factor):
        scores = [s for _, s in scored_docs]
        threshold = np.mean(scores) - std_factor * np.std(scores)
        return [(doc, s) for doc, s in scored_docs if s >= threshold]

    @staticmethod
    def score_gap(scored_docs, min_docs=2):
        if len(scored_docs) <= min_docs:
            return scored_docs
        scores = [s for _, s in scored_docs]
        # 연속 점수 간 차이를 계산해 가장 큰 갭의 위치를 찾음
        gaps = [(scores[i] - scores[i+1], i+1) for i in range(len(scores)-1)]
        max_gap_idx = max(gaps, key=lambda x: x[0])[1]
        cut = max(min_docs, max_gap_idx)
        return scored_docs[:cut]
```

#### AdaptiveFilter — 자동 전략 선택

문서 점수 분포를 분석해 최적 전략을 자동 선택.

```python
class AdaptiveFilter:
    @staticmethod
    def apply(scored_docs):
        scores = [s for _, s in scored_docs]
        std = np.std(scores)
        score_range = max(scores) - min(scores)

        if std > 0.2:          # 점수 편차가 크면 → Score Gap
            return ScoreFilter.score_gap(scored_docs)
        elif score_range < 0.1: # 점수가 밀집되면 → Top Half (상위 50%)
            cut_idx = max(1, len(scored_docs) // 2)
            return scored_docs[:cut_idx]
        else:                   # 그 외 → Dynamic Threshold
            return ScoreFilter.dynamic_threshold(scored_docs, std_factor=1.0)
```

### 5-3. 문서 압축 (Context Compression)

검색 문서 전체를 LLM에 넘기면 토큰 비용 증가 + 노이즈 증가.
질문 관련 핵심 내용만 추출해 컨텍스트 품질을 높이는 과정.

| 방식 | 설명 | 특징 |
|---|---|---|
| Extractive | 쿼리 키워드 겹침 기반 문장 선택 | 빠름, 단순, API 비용 없음 |
| LLM 기반 | LLM이 관련 문장만 추출 | 높은 정확도, API 비용 발생 |
| MMR | 관련성 + 다양성 동시 최적화 | 중복 제거, 임베딩 필요 |

#### Extractive Compression

```python
def extractive_compress(query, document, max_sentences=3):
    sentences = re.split(r'[.!?]\s*', document)  # 문장 단위로 분리
    query_terms = set(query.lower().split())       # 쿼리 단어 집합
    scored = []
    for sent in sentences:
        sent_terms = set(sent.lower().split())
        overlap = len(query_terms & sent_terms)    # 쿼리와의 단어 겹침 수
        scored.append((sent, overlap))
    scored.sort(key=lambda x: x[1], reverse=True)  # 겹침 많은 순 정렬
    selected = [s for s, _ in scored[:max_sentences]]
    return ". ".join(selected) + "."
```

#### MMR 기반 압축 (Maximum Marginal Relevance)

관련성(Relevance)과 다양성(Diversity)을 동시에 최적화.

```
MMR(s) = λ × Relevance(s, query) - (1-λ) × max Similarity(s, selected)
```

- `λ` 높음 (0.7) → 관련성 우선 (정보량 집중)
- `λ` 낮음 (0.3) → 다양성 우선 (중복 제거)

```python
def mmr_compress(query, document, max_sentences=3, param=0.5):
    sentences = re.split(r'[.!?]\s*', document)
    sentences = [s.strip() for s in sentences if len(s.strip()) > 10]
    query_emb = get_embedding(query)
    sent_embs = [get_embedding(s) for s in sentences]

    selected = []
    remaining = list(range(len(sentences)))
    for _ in range(max_sentences):
        best_score = -float('inf')
        best_idx = -1
        for idx in remaining:
            relevance = cosine_sim(query_emb, sent_embs[idx])    # 쿼리와의 관련성
            if selected:
                max_sim = max(cosine_sim(sent_embs[idx], sent_embs[s]) for s in selected)
            else:
                max_sim = 0.0
            mmr = param * relevance - (1 - param) * max_sim      # MMR 계산
            if mmr > best_score:
                best_score = mmr
                best_idx = idx
        selected.append(best_idx)
        remaining.remove(best_idx)
    return ". ".join(sentences[i] for i in sorted(selected)) + "."
```

#### 압축 품질 평가

| 지표 | 설명 |
|---|---|
| 압축률 | `len(compressed) / len(original)` — 낮을수록 많이 압축 |
| 키워드 보존율 | 쿼리 단어가 압축 후에도 남아있는 비율 |
| 의미 유사도 | 원본과 압축본의 임베딩 코사인 유사도 |

```python
def evaluate_compression(original, compressed, query):
    ratio = len(compressed) / len(original)                        # 1) 압축률
    query_terms = set(query.lower().split())
    orig_terms = query_terms & set(original.lower().split())
    comp_terms = query_terms & set(compressed.lower().split())
    keyword_coverage = len(comp_terms) / len(orig_terms)           # 2) 키워드 보존율
    original_emb = get_embedding(original)
    compressed_emb = get_embedding(compressed)
    semantic_similarity = cosine_sim(original_emb, compressed_emb) # 3) 의미 유사도
    return {'compression_ratio': ratio, 'keyword_coverage': keyword_coverage,
            'semantic_similarity': semantic_similarity}

### 5-4. 검색 품질 평가 지표 — AP, mAP, ILS

재순위화·필터링이 실제로 검색 품질을 개선했는지 정량 측정.

#### AP (Average Precision)

각 관련 문서 위치에서의 Precision 평균. 1에 가까울수록 관련 문서가 상위에 밀집.

```
예시: 검색 결과 = [관련O, 관련X, 관련O, 관련X, 관련O]
P@k:              1/1     -     2/3     -     3/5
AP = (1/1 + 2/3 + 3/5) / 3 = 0.756
```

```python
def average_precision(retrieved_ids, relevant_ids):
    relevant_set = set(relevant_ids)
    hits = 0
    sum_precision = 0.0
    for i, doc_id in enumerate(retrieved_ids):
        if doc_id in relevant_set:
            hits += 1
            precision_at_i = hits / (i + 1)  # 현재 위치까지의 Precision
            sum_precision += precision_at_i
    return sum_precision / len(relevant_set) if relevant_set else 0.0
```

#### mAP (Mean Average Precision)

여러 질문에 대한 AP의 평균 → 시스템 전체 성능을 하나의 수치로 표현.

```python
def mean_average_precision(query_results):
    # query_results = {query1: (retrieved_ids, relevant_ids), query2: ...}
    aps = []
    for query, (retrieved, relevant) in query_results.items():
        ap = average_precision(retrieved, relevant)
        aps.append(ap)
    return np.mean(aps)
```

#### ILS (Intra-List Similarity)

검색 결과 내 문서들 간의 상호 유사도. 다양성 평가 지표.

- ILS 낮음 → 결과가 다양 (사용자에게 다양한 정보 제공)
- ILS 높음 → 결과가 유사 (중복 위험)

```python
def intra_list_similarity(doc_ids, doc_embeddings, top_k=5):
    ids = doc_ids[:top_k]
    embs = [doc_embeddings[did] for did in ids if did in doc_embeddings]
    if len(embs) < 2:
        return 0.0
    sims = []
    for i in range(len(embs)):
        for j in range(i+1, len(embs)):  # 모든 문서 쌍의 코사인 유사도
            sims.append(cosine_sim(embs[i], embs[j]))
    return np.mean(sims)
```

| 지표 | 핵심 질문 | 특징 |
|---|---|---|
| AP | "관련 문서가 상위에 있나?" | 관련 문서 위치 반영, 단일 쿼리 |
| mAP | "시스템 전체가 잘 작동하나?" | AP의 평균, 여러 쿼리 |
| ILS | "결과가 다양한가?" | 정확도가 아닌 다양성 측정 |

### 5-5. A/B Test — t-test와 Cohen's d

기존 방식(A)과 개선 방식(B)의 성능 차이를 통계적으로 검증.

**통계 검증 기본 구조:**
- 귀무가설 (H₀): A, B 차이 없음
- 대립가설 (H₁): A, B 차이 있음
- `p_value < alpha(0.05)` → 귀무가설 기각 → **유의미한 차이 존재**

**Cohen's d — 효과 크기:**

| 값 | 해석 |
|---|---|
| d < 0.2 | 작은 효과 |
| 0.2 ≤ d < 0.5 | 중간 효과 |
| 0.5 ≤ d < 0.8 | 큰 효과 |
| d ≥ 0.8 | 매우 큰 효과 |

```python
def ab_test(scores_a, scores_b, alpha=0.05):
    # 대응표본 t-검정: 같은 쿼리에 대한 A/B 점수를 비교
    t_stat, p_val = stats.ttest_rel(scores_a, scores_b)

    mean_a = np.mean(scores_a)
    mean_b = np.mean(scores_b)

    # Cohen's d: 효과 크기 측정 (pooled 표준편차 사용)
    pooled_std = np.sqrt((np.std(scores_a)**2 + np.std(scores_b)**2) / 2)
    cohens_d = (mean_a - mean_b) / pooled_std if pooled_std > 0 else 0.0

    return {
        'mean_A': mean_a, 'mean_B': mean_b,
        'p_val': p_val, 'cohens_d': cohens_d,
        'significant': p_val < alpha  # True이면 유의미한 차이
    }
```

**실행 흐름:**

```python
# 각 쿼리에 대해 원본/재순위화 AP를 계산
aps_before = []   # 벡터 검색 AP들
aps_after = []    # BM25 재순위화 후 AP들
for q, relevant in test_queries.items():
    results = vector_search(q, vectorstore)
    orig_order = [doc.metadata['id'] for doc, _ in results]
    aps_before.append(average_precision(orig_order, relevant))
    bm25_results = bm25_reranker.rerank(q, results)
    bm25_order = [doc.metadata['id'] for doc, _ in bm25_results]
    aps_after.append(average_precision(bm25_order, relevant))

result = ab_test(aps_before, aps_after)
# → p_val < 0.05이고 cohens_d > 0.5이면 재순위화가 유의미하게 성능 향상
```

### 5-6. LLM-as-Judge — Pointwise, Pairwise, Reference-based

#### 기존 지표의 한계

| 지표 | 방식 | 한계 |
|---|---|---|
| BLEU | n-gram 겹침 (정밀도 중심) | 의미적 동의어 무시, 창의적 표현 불이익 |
| ROUGE | n-gram 겹침 (재현율 중심) | 동일한 한계, 어순 무관 |

```python
def simple_rouge_f1(reference, candidate):
    ref_words = Counter(reference.split())
    cand_words = Counter(candidate.split())
    overlap = sum(min(ref_words[w], cand_words.get(w, 0)) for w in ref_words)
    r = overlap / sum(ref_words.values()) if ref_words else 0  # 재현율
    p = overlap / sum(cand_words.values()) if cand_words else 0  # 정밀도
    return 2 * p * r / (p + r) if (p + r) > 0 else 0  # F1
```

#### LLM-as-Judge 방식 비교

| 방식 | 입력 | 출력 | 용도 |
|---|---|---|---|
| Pointwise | 질문 + 답변 1개 | 1~5 점수 | 절대적 품질 측정 |
| Pairwise | 질문 + 답변 2개 | A/B/Tie | 상대 비교 |
| Reference-based | 질문 + 정답 + 답변 | 1~5 점수 + 상세 분석 | 정답이 있을 때 |

#### 앙상블 평가 (ensemble_judge)

여러 평가 함수의 점수를 통합해 편향을 줄이고 안정성 향상.

| 집계 방식 | 설명 |
|---|---|
| mean | 단순 평균 |
| median | 중앙값 (이상치에 강건) |
| trimmed mean | 최대/최소 제거 후 평균 |
| weighted mean | 가중 평균 (신뢰도 높은 함수에 가중치) |

```python
def ensemble_judge(question, answer, judge_fns, weights=None):
    scores = []
    for name, fn in judge_fns:
        result = fn(question, answer)  # 각 평가 함수 실행
        scores.append(result.get('score'))

    mean_score = np.mean(scores)
    median_score = np.median(scores)

    # 절삭평균: 최대값, 최소값 제거 후 평균
    if len(scores) >= 3:
        trimmed_scores = sorted(scores)[1:-1]
        trimmed_mean = np.mean(trimmed_scores)

    # 가중평균: weights에 따라 스코어 반영 비율 조절
    if weights is not None:
        weighted_mean = np.average(scores, weights=weights)

    return {'mean_score': mean_score, 'median_score': median_score,
            'trimmed_score': trimmed_mean, 'weighted_score': weighted_mean}

# 사용 예시
judge_fns = {("pointwise", pointwise_judge), ("Few-shot", fewshot_judge)}
result = ensemble_judge(question, answer, judge_fns, weights=[0.6, 0.4])
```

### 5-8. 평가 편향 & 검증 (Bias & Validation)

LLM 기반 평가 시 주의해야 할 편향:
1. **길이 편향**: 긴 답변에 더 높은 점수를 주는 경향
2. **위치 편향**: 입력에서 앞에 등장하는 텍스트에 더 주의를 기울임 (Lost-in-the-middle)
3. **자기 강화 편향**: LLM이 자기가 생성한 답변에 더 높은 점수 부여

#### 길이 편향 검출 (Length Bias)

```python
def detect_length_bias(question, short_answer, long_answer):
    short_result = pointwise_judge(question, short_answer)
    long_result = pointwise_judge(question, long_answer)
    bias = long_result.get('score') - short_result.get('score')
    return {
        'short_score': short_result.get('score'),
        'long_score': long_result.get('score'),
        'bias': bias,
        'has_bias': bias > 0  # 양수면 길이 편향 존재 가능
    }
```

#### 위치 편향 검출 (Position Bias)

- `pairwise_judge_with_swap`: 같은 질문을 (A→B), (B→A) 순서로 두 번 평가
- `detect_position_bias`: n_trials회 반복 → `inconsistent / total = bias_rate`
- 편향률이 높을수록 → 평가가 내용보다 위치에 의존

#### 재현성 테스트 (Reproducibility)

```python
def test_reproductbility(question, answer, n_trials=5):
    scores = []
    for i in range(n_trials):
        result = pointwise_judge(question, answer)
        scores.append(result.get('score'))  # 동일 입력 반복 평가

    mean = np.mean(scores)
    std = np.std(scores)
    cv = std / mean if mean > 0 else float('inf')  # 변동계수(CV)
    return {'scores': scores, 'mean': mean, 'std': std, 'cv': cv}
    # CV가 낮을수록 평가가 안정적
```

#### 판별력 테스트 (Discriminability)

```python
def test_discriminability(question, good_answer, bad_answer, n_trials=3):
    good_scores = []
    bad_scores = []
    for _ in range(n_trials):
        good_scores.append(pointwise_judge(question, good_answer).get('score'))
        bad_scores.append(pointwise_judge(question, bad_answer).get('score'))

    gap = np.mean(good_scores) - np.mean(bad_scores)
    return {
        'good_mean': np.mean(good_scores),

    │
    ├─ vectorstore = FAISS.from_documents(documents, embeddings_model)
    │   └─ vector_search(query, vectorstore) → [(doc, similarity_score), ...]
    │       * FAISS는 임베딩 벡터 간 거리를 계산 → 1/(1+distance)로 유사도 변환
    │
    ├─ bm25_retriever = BM25Retriever.from_documents(documents)
    │   └─ bm25_retriever.invoke(query) → [doc, ...]
    │       * BM25는 키워드 매칭 기반 → TF-IDF 개선 알고리즘
    │
    └─ doc_embeddings = {doc_id: embedding_vector}
        └─ ILS 계산 시 사용 (문서 간 유사도 비교)
```

### Q&A: bm25는 vectorstore와 다른 건가?

> **다름.** 같은 `documents` 리스트에서 출발하지만, 데이터 처리 방식이 완전히 다름.

| | vectorstore (FAISS) | bm25_retriever |
|---|---|---|
| 데이터 소스 | 동일한 `documents` 리스트 | 동일한 `documents` 리스트 |
| 생성 방법 | `FAISS.from_documents(documents, embeddings_model)` | `BM25Retriever.from_documents(documents)` |
| 내부 저장 | 각 문서의 **임베딩 벡터** (숫자 배열) | 각 문서의 **토큰 목록** (단어 리스트) |
| 검색 방식 | 쿼리도 임베딩 → 벡터 간 거리 계산 | 쿼리를 토큰화 → TF-IDF/BM25 점수 계산 |
| 강점 | 의미적 유사도 (동의어, 유사 표현) | 정확한 키워드 매칭 (고유명사, 전문용어) |

```python
# 두 객체 모두 같은 documents에서 만들어짐
documents = [Document(page_content="...", metadata={"id": "d1"}), ...]

# FAISS: 문서 → 임베딩 벡터로 변환 → 벡터 인덱스에 저장
vectorstore = FAISS.from_documents(documents, embeddings_model)
# 내부: {"d1": [0.12, -0.34, 0.56, ...], "d2": [...], ...}

# BM25: 문서 → 단어로 토큰화 → 단어 빈도 통계 저장
bm25_retriever = BM25Retriever.from_documents(documents, k=5)
# 내부: {"d1": ["트랜스포머", "self-attention", ...], "d2": [...], ...}
```

## 고려사항 & 질문 모음

1. **재순위화 방식 선택 기준**: 키워드 중심 도메인(법률, 의료)에서는 BM25가 효과적, 의미 파악이 중요한 도메인에서는 LLM Reranking 유리. Hybrid는 범용적이나 비용 증가.

2. **BM25 vs FAISS의 근본 차이**: 같은 문서에서 출발하지만 저장 형태가 다름(토큰 vs 벡터). "트랜스포머"라는 단어가 정확히 있어야 하면 BM25, "딥러닝 구조"처럼 유사한 의미로 찾으려면 FAISS.

3. **Score Filtering의 순서**: 반드시 재순위화 **이후**에 수행. 재순위화 전에 필터링하면 잠재적으로 관련 있는 문서를 조기 제거할 위험.

4. **MMR의 λ 선택**: 정보가 하나만 필요하면 λ=0.7~0.9 (관련성 우선), 여러 관점이 필요하면 λ=0.3~0.5 (다양성 우선).

5. **LLM-as-Judge의 한계**: LLM이 자기 자신을 평가하면 자기 강화 편향이 발생할 수 있음. 다른 모델로 교차 검증하거나 앙상블을 사용해 완화.

6. **평가 시스템 검증이 필요한 이유**: 평가 도구 자체가 불안정하면 A/B 테스트 결과를 신뢰할 수 없음. CV(변동계수)로 재현성을, gap으로 판별력을 확인해야 함.

7. **Few-shot Judge의 예시 설계**: 좋은 예시(4점)와 나쁜 예시(2점)를 함께 제공하면 LLM이 채점 기준을 더 명확히 학습. 예시의 질이 평가 일관성에 직접 영향.
