## 핵심 개념 상세 정리

### 4-1. 검색 평가 메트릭 1 — Hit Rate@K, MRR, DCG/nDCG

#### Hit Rate@K

K개 검색 결과 안에 정답 문서가 **하나라도** 포함되면 1, 없으면 0. 가장 단순한 이진 메트릭.

```python
def hit_rate_at_k(query_id, k):
    qa = next(q for q in qa_dataset if q['query_id'] == query_id)
    relevant_ids = set(qa['relevant_doc_ids'])
    retrieved = search_results_cache[query_id][:k]
    retrieved_ids = {r['doc_id'] for r in retrieved}
    return 1 if relevant_ids & retrieved_ids else 0

def average_hit_rate_at_k(k):
    hits = [hit_rate_at_k(qa['query_id'], k) for qa in qa_dataset]
    return sum(hits) / len(hits)
```

#### MRR (Mean Reciprocal Rank)

첫 번째 정답이 **몇 번째 위치**에 있는지의 역수를 평균. 1위 = 1.0, 2위 = 0.5, 3위 ≈ 0.33.

```python
def reciprocal_rank(query_id, k):
    qa = next(q for q in qa_dataset if q['query_id'] == query_id)
    relevant_ids = set(qa['relevant_doc_ids'])
    retrieved = search_results_cache[query_id][:k]

    for rank, result in enumerate(retrieved, 1):
        if result['doc_id'] in relevant_ids:
            return 1.0 / rank
    return 0.0  # K 안에 정답 없으면 1/∞ = 0

def mean_reciprocal_rank(k):
    rr_scores = [reciprocal_rank(qa['query_id'], k) for qa in qa_dataset]
    return sum(rr_scores) / len(rr_scores)
```

#### DCG / nDCG

위치에 따라 감쇠되는 가중치를 적용한 메트릭. nDCG는 이상적 순서(IDCG)로 정규화.

```
DCG@K = Σ rel(i) / log2(i+2)   (i=0부터, rel=1이면 관련, 0이면 무관)
nDCG@K = DCG@K / IDCG@K        (IDCG = 관련 문서가 앞에 다 몰려있을 때의 DCG)
```

```python
def dcg_at_k(relevances, k):
    relevances = relevances[:k]
    return sum(rel / np.log2(i + 2) for i, rel in enumerate(relevances))

def ndcg_at_k(query_id, k):
    qa = next(q for q in qa_dataset if q['query_id'] == query_id)
    relevant_ids = set(qa['relevant_doc_ids'])
    retrieved = search_results_cache[query_id][:k]

    relevances = [1 if r['doc_id'] in relevant_ids else 0 for r in retrieved]
    dcg = dcg_at_k(relevances, k)

    # 이상적 순서: 관련 문서를 앞에 배치
    total_relevant = len(relevant_ids)
    ideal_relevances = [1] * min(total_relevant, k) + [0] * max(0, k - total_relevant)
    idcg = dcg_at_k(ideal_relevances, k)

    return dcg / idcg if idcg > 0 else 0.0
```

| 메트릭 | 핵심 질문 | 특징 |
|--------|-----------|------|
| Hit Rate@K | "K개 안에 정답이 있나?" | 이진값, 위치 무관 |
| MRR | "첫 정답이 몇 위?" | 위치 반영, 첫 정답만 봄 |
| nDCG | "관련 문서가 앞에 있나?" | 위치 + 관련도 모두 반영 |

### 4-2. 검색 평가 메트릭 2 — Precision/Recall/F1, BLEU, LLM-as-judge

> 📄 강의 자료: `evaluate.pdf` (RAG 평가 프레임워크 — Precision & Recall 섹션)

#### TP/FP/FN/TN 혼동 행렬

| | 실제 관련 | 실제 무관 |
|-|---------|---------|
| **검색됨** | TP (맞게 포착) ✓ | FP (노이즈) ✗ |
| **미검색** | FN (누락) ✗ | TN (올바른 제외) ✓ |

💡 TN은 수백만 개로 의미 없으므로 실무에서 Precision/Recall 중심으로 평가.

#### Precision@K, Recall@K, F1, R-Precision, F-beta

```python
def precision_at_k(query_id, k):
    qa = next(q for q in qa_dataset if q['query_id'] == query_id)
    relevant_ids = set(qa['relevant_doc_ids'])
    retrieved_ids = {r['doc_id'] for r in search_results_cache[query_id][:k]}
    return len(relevant_ids & retrieved_ids) / k

def recall_at_k(query_id, k):
    qa = next(q for q in qa_dataset if q['query_id'] == query_id)
    relevant_ids = set(qa['relevant_doc_ids'])
    retrieved_ids = {r['doc_id'] for r in search_results_cache[query_id][:k]}
    return len(relevant_ids & retrieved_ids) / len(relevant_ids) if relevant_ids else 0.0

def f1_at_k(query_id, k):
    p, r = precision_at_k(query_id, k), recall_at_k(query_id, k)
    return 2 * p * r / (p + r) if (p + r) > 0 else 0.0

def r_precision(query_id):
    """각 쿼리의 관련 문서 수 R을 K로 사용 → 쿼리 간 공정 비교"""
    qa = next(q for q in qa_dataset if q['query_id'] == query_id)
    R = len(set(qa['relevant_doc_ids']))
    return precision_at_k(query_id, R) if R > 0 else 0.0

def f_beta_at_k(query_id, k, beta):
    """β < 1: Precision 중시 (RAG) | β = 1: F1 | β > 1: Recall 중시 (의료/법률)"""
    p, r = precision_at_k(query_id, k), recall_at_k(query_id, k)
    b2 = beta ** 2
    return (1 + b2) * p * r / (b2 * p + r) if (b2 * p + r) > 0 else 0.0
```

**Precision↔Recall Trade-off**

```
임계값 높이면 → 반환 문서 수 ↓ → Precision↑, Recall↓
임계값 낮추면 → 반환 문서 수 ↑ → Recall↑, Precision↓

K↑ → 더 많은 문서 포함 → Recall↑, Precision↓ (노이즈 유입)
```

⚠️ **Precision 낮으면**: LLM에 노이즈 청크 전달 → 환각(Hallucination) 유발
⚠️ **Recall 낮으면**: 핵심 정보 누락 → 불완전한 답변 생성

**F-beta 실무 가이드**

| β 값 | 가중치 | 사용 시나리오 |
|------|--------|-------------|
| β = 0.5 | Precision 2배 중시 | RAG 컨텍스트, 스팸 필터 |
| β = 1.0 | 동일 (= F1) | 일반 웹 검색, 추천 시스템 |
| β = 2.0 | Recall 2배 중시 | 의료·법률 문서 검색 |

**R-Precision의 흥미로운 성질**: R-Precision = TP/R = TP/(TP+FN) = Recall@R  
수식은 같지만, 분모를 K로 보는가(Precision 느낌) vs R로 보는가(Recall 느낌)의 관점 차이.

#### BLEU Score — 생성 품질 평가

전통 NLP 메트릭. API 호출 없이 저렴하고 재현 가능하지만, 정답 텍스트가 필요.

```python
from collections import Counter

def get_ngrams(tokens, n):
    return Counter(tuple(tokens[i:i+n]) for i in range(len(tokens) - n + 1))

def bleu_score(reference, hypothesis, max_n=4):
    ref_tokens = reference.split()
    hyp_tokens = hypothesis.split()
    c, r = len(hyp_tokens), len(ref_tokens)
    # Brevity Penalty: 짧은 생성문 패널티
    bp = 1.0 if c >= r else (np.exp(1 - r/c) if c > 0 else 0.0)

    precisions = {}
    for n in range(1, max_n + 1):
        ref_ngrams = get_ngrams(ref_tokens, n)
        hyp_ngrams = get_ngrams(hyp_tokens, n)
        if not hyp_ngrams:
            precisions[n] = 0.0
            continue
        clipped = sum(min(cnt, ref_ngrams.get(ng, 0)) for ng, cnt in hyp_ngrams.items())
        precisions[n] = clipped / sum(hyp_ngrams.values())

    log_avg = sum(np.log(p) for p in precisions.values() if p > 0) / max_n
    return bp * np.exp(log_avg)
```

### 4-3. TF-IDF와 BM25 — 키워드 검색의 이론적 기반

> 📄 강의 자료: `evaluate.pdf` (BM25 알고리즘 이해와 구현 섹션)

#### TF-IDF

```
TF(t, d) = f(t, d) / |d|             # 문서 d에서 단어 t의 빈도 비율
IDF(t)   = log(N / df(t))            # 전체 N 문서 중 t 등장 문서 수의 역수
TF-IDF(t, d) = TF(t, d) × IDF(t)    # 곱 → "이 문서에서 이 단어가 얼마나 독특하게 중요한가"
```

| TF | IDF | TF-IDF | 해석 |
|----|-----|--------|------|
| 높음 | 낮음 | 중간 | "은/는" 같은 조사 — 흔하지만 이 문서엔 많음 |
| 높음 | 높음 | **높음** ⭐ | 이 문서의 핵심 키워드 — 희귀하고 많이 나옴 |
| 낮음 | 높음 | 중간 | 부수적 언급 — 희귀하지만 한두 번만 나옴 |
| 낮음 | 낮음 | 낮음 | 의미 없는 단어 |

**TF-IDF의 두 가지 한계:**
1. **TF 포화 없음**: "Python" 5번 나온 문서와 50번 나온 문서는 관련도가 비슷한데, TF는 10배 차이로 계산
2. **문서 길이 미보정**: 1,000단어 문서에서 5번 ≠ 100단어 문서에서 5번 (의미가 완전 다름)

```python
class TFIDF:
    def __init__(self, documents):
        self.docs = documents
        self.tokenized = [doc.lower().split() for doc in documents]
        self.N = len(documents)
        self.df = {}
        for tokens in self.tokenized:
            for t in set(tokens):
                self.df[t] = self.df.get(t, 0) + 1

    def tf(self, term, doc_tokens):
        return doc_tokens.count(term) / len(doc_tokens)

    def idf(self, term):
        return math.log(self.N / self.df.get(term, 1))

    def score(self, term, doc_idx):
        return self.tf(term, self.tokenized[doc_idx]) * self.idf(term)
```

#### BM25 — TF-IDF의 두 가지 한계 해결

```
BM25(q, d) = Σ IDF(t) × [f(t,d) × (k₁+1)] / [f(t,d) + k₁ × (1 - b + b × |d|/avgdl)]
              t∈q
```

- **IDF 평활화**: `log((N - df + 0.5) / (df + 0.5) + 1)` — 극단값 방지
- **k₁ (TF 포화)**: TF가 아무리 커도 `k₁+1`에 수렴 → 단어 폭탄 방지. 기본값 1.5
- **b (길이 정규화)**: `1 - b + b × |d|/avgdl`. b=0이면 길이 무시, b=1이면 완전 정규화. 기본값 0.75

```python
class BM25:
    def __init__(self, documents, k1=1.5, b=0.75):
        self.k1, self.b = k1, b
        self.docs = documents
        self.tokenized = [doc.lower().split() for doc in documents]
        self.N = len(documents)
        self.avgdl = sum(len(d) for d in self.tokenized) / self.N
        self.df = {}
        for tokens in self.tokenized:
            for t in set(tokens):
                self.df[t] = self.df.get(t, 0) + 1

    def idf(self, term):
        df = self.df.get(term, 0)
        return math.log((self.N - df + 0.5) / (df + 0.5) + 1)

    def score(self, query, doc_idx):
        tokens = self.tokenized[doc_idx]
        dl = len(tokens)
        return sum(
            self.idf(t) * (freq * (self.k1 + 1)) /
            (freq + self.k1 * (1 - self.b + self.b * dl / self.avgdl))
            for t in query.lower().split()
            if (freq := tokens.count(t)) > 0
        )
```

### 4-4. 하이브리드 검색 — BM25 + FAISS + RRF

#### 키워드 검색 vs. 벡터 검색

| 비교 항목 | 키워드 검색 (BM25) | 벡터 검색 (FAISS) |
|---------|-----------------|-----------------|
| 점수 범위 | 0 ~ 무한대 (상한 없음) | 보통 0.0 ~ 1.0 |
| 강점 | 고유명사, 신조어, 약품명 등 | 의미 유사성, 다국어 |
| 약점 | 어휘 불일치(vocabulary mismatch) | 새로운 용어 취약 |

💡 두 점수를 단순 합산하면 BM25 점수(예: 150)가 코사인 유사도(예: 0.8)를 압도 → 정규화 필요

#### EnsembleRetriever — 점수 기반 가중 합산

```python
from langchain_community.vectorstores import FAISS
from langchain.retrievers import EnsembleRetriever

vectorstore = FAISS.from_documents(docs_lc, embeddings)
vector_retriever = vectorstore.as_retriever(search_kwargs={"k": 3})

ensemble = EnsembleRetriever(
    retrievers=[vector_retriever, bm25_retriever],
    weights=[0.5, 0.5],  # α: 벡터 가중치, (1-α): BM25 가중치
)
results = ensemble.invoke("Python 데이터 과학")
```

#### RRF (Reciprocal Rank Fusion) — 순위 기반 통합

점수를 버리고 **순위(Rank)**로 변환하여 공정하게 합산. 이상치에 robust.

```
RRF 점수 = Σ 1 / (k + rank_i)    # k=60이 표준, k가 크면 순위 차이 희석
```

```python
def rrf(rankings, k=60):
    """rankings: [(doc_id, score), ...] 형태의 리스트들"""
    rrf_scores = {}
    for ranking in rankings:
        for rank, (doc_id, _) in enumerate(ranking, start=1):
            rrf_scores[doc_id] = rrf_scores.get(doc_id, 0) + 1 / (k + rank)
    return sorted(rrf_scores.items(), key=lambda x: x[1], reverse=True)

# 사용 예시
bm25_results = bm25.search(query, top_k=5)   # [(doc_id, score), ...]
vector_results = vector_search(query, ...)
combined = rrf([bm25_results, vector_results], k=60)
```

#### α 가중치 그리드 서치

```python
def hybrid_search(query, w_bm25=0.5, top_k=5, k=60):
    bm25_r = bm25.search(query, top_k=top_k)
    vec_r = vector_search(query, documents, doc_embs, top_k=top_k)

    rrf_scores = {}
    for rank, (doc_id, _) in enumerate(bm25_r, start=1):
        rrf_scores[doc_id] = rrf_scores.get(doc_id, 0) + w_bm25 / (k + rank)
    for rank, (doc_id, _) in enumerate(vec_r, start=1):
        rrf_scores[doc_id] = rrf_scores.get(doc_id, 0) + (1 - w_bm25) / (k + rank)

    return sorted(rrf_scores.items(), key=lambda x: x[1], reverse=True)[:top_k]

# 최적 α 탐색
weights = np.arange(0, 1.05, 0.1)
results = []
for w in weights:
    avg_mrr = np.mean([mrr(hybrid_search(item['query'], w_bm25=w), item['relevant'])
                       for item in eval_dataset])
    results.append((w, avg_mrr))
best_w, best_mrr = max(results, key=lambda x: x[1])
```

### 4-6. 쿼리 확장 — Multi-Query와 HyDE

> 📄 참고 논문: ACL 2023 — HyDE (Hypothetical Document Embeddings): https://aclanthology.org/2023.acl-long.99.pdf

벡터 검색의 **어휘 불일치 문제(vocabulary mismatch)**를 해결하는 쿼리 변환 기법들.

#### Multi-Query Retriever

원본 쿼리를 LLM을 이용해 **여러 관점**으로 재작성한 후, 각각 독립 검색 → RRF로 통합.

```
원본: "원자로 출력 제어 방법"
├─ "핵분열 연쇄반응 조절 기술"       ← 기술 관점
├─ "제어봉 삽입 및 냉각재 유량 조정" ← 운전 관점
└─ "Xenon 진동 억제 전략"           ← 물리 관점
```

```python
MULTI_QUERY_PROMPT = """주어진 질문을 3가지 서로 다른 관점에서 재작성하세요.
각 쿼리는 원래 질문의 의도를 유지하되, 다른 단어와 표현을 사용하세요.

원래 질문: {query}

재작성1:
재작성2:
재작성3:"""

def generate_multi_queries(query, n=3):
    response = llm.invoke(MULTI_QUERY_PROMPT.format(query=query)).content
    queries = [line.strip() for line in response.strip().split('\n') if line.strip()]
    return queries[:n]

def multi_query_search(query, top_k=3):
    expanded = generate_multi_queries(query, n=3)
    all_queries = [query] + expanded
    return search_with_rrf(all_queries, top_k=top_k)

def search_with_rrf(queries, top_k=3, k=60):
    rrf_scores = {}
    for q in queries:
        results = search_faiss(q, top_k=top_k)
        for rank, (doc_id, _) in enumerate(results, 1):
            rrf_scores[doc_id] = rrf_scores.get(doc_id, 0) + 1 / (k + rank)
    return sorted(rrf_scores.items(), key=lambda x: x[1], reverse=True)[:top_k]
```

**쿼리 다양성 측정:**

```python
def query_diversity(queries):
    """평균 코사인 유사도의 역수로 다양성 측정. 높을수록 다양."""
    embs = np.array(embeddings.embed_documents(queries))
    norms = np.linalg.norm(embs, axis=1, keepdims=True)
    emb_norms = embs / norms

    sims = [np.dot(emb_norms[i], emb_norms[j])
            for i in range(len(queries)) for j in range(i+1, len(queries))]
    return 1 - np.mean(sims)  # 1 = 완전히 다양, 0 = 동일
```

#### HyDE (Hypothetical Document Embeddings)

쿼리를 직접 임베딩하는 대신, LLM이 **가상의 정답 문서**를 먼저 생성하고 그것을 임베딩.

```
원본 쿼리 → [LLM] → 가상 답변 문서 생성 → [Embedding] → 벡터 검색
```

```python
HYDE_PROMPT = """아래 질문에 대해 3-4문장으로 답변을 작성해주세요.
정확하지 않아도 괜찮습니다. 관련 주제의 문서처럼 작성하세요.

질문: {query}

답변:"""

def hyde_search(query, top_k=3):
    # 가상 문서 생성
    hypothesis = llm.invoke(HYDE_PROMPT.format(query=query)).content
    # 가상 문서를 임베딩하여 검색
    hyp_emb = embeddings.embed_query(hypothesis)
    return search_by_vector(hyp_emb, top_k)
```

| 기법 | 원리 | 강점 | 약점 |
|------|------|------|------|
| 기본 벡터 검색 | 쿼리 임베딩 직접 비교 | 빠름 | 어휘 불일치 취약 |
| Multi-Query | LLM이 N개 관점으로 재작성 | Recall 향상 | LLM 호출 N배 |
| HyDE | LLM이 가상 답변 생성 후 임베딩 | 문서-쿼리 공간 불일치 해소 | 가상 답변 품질 의존 |

## 고려사항 & 질문 모음

1. **메트릭 선택 기준**: RAG 시스템에서는 왜 Precision이 Recall보다 더 중요한가?
   - 노이즈 문서가 컨텍스트로 들어가면 LLM이 환각을 일으킬 수 있기 때문.
   - "잘못 넣는 것이 못 넣는 것보다 더 해롭다" (F-beta에서 β < 1 사용)

2. **RRF의 k=60 표준값**: k 값이 커질수록 순위 차이가 희석됨 (1등과 2등의 차이 감소). 이상치에 robust해지는 대신 1등의 가중치가 약해짐.

3. **HyDE의 한계**: LLM이 생성한 가상 답변의 품질에 의존함. 답변이 완전히 엉뚱하면 오히려 성능 저하.

4. **한국어 BM25**: `BM25Retriever`는 기본적으로 공백 기반 토크나이저 사용. 한국어에서는 `kiwipiepy` 등으로 커스텀 토크나이저를 주입해야 성능이 제대로 나옴.

5. **검색 평가 데이터셋 구성**: `qa_dataset`의 `relevant_doc_ids`가 실제 문서 DB에 존재하는지 검증하는 `validate_dataset()` 함수의 중요성 — 데이터 누락 시 Recall이 왜곡됨.

6. **`next()` vs 리스트 컴프리헨션**: `qa = next(q for q in qa_dataset if q['query_id'] == query_id)` 패턴은 찾자마자 반환하는 iterator이므로, 리스트를 전부 만드는 것보다 효율적.
